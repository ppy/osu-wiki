# Code of conduct for modding and mapping

**code of conduct for modding and mapping** คือชุดกฎและแนวทางที่ใช้กับ ecosystem ของ [modding](/wiki/Modding) และ [mapping](/wiki/Beatmapping) ใน osu! ทั้งหมด และเป็นส่วนขยายของ[กฎคอมมูนิตี้ osu!](/wiki/Rules) เพื่อให้การพูดคุยเรื่อง mapping และ modding ใน threads เกิดขึ้นในสภาพแวดล้อมที่สร้างสรรค์ เป็นบวก และมี productivity code of conduct จึงสำคัญต่อการทำให้ทุกคนเข้าใจตรงกัน เมื่อเข้าร่วม ecosystem ของ modding และ mapping จำเป็นต้องทำตาม code of conduct นี้ พฤติกรรมไม่เหมาะสมที่ละเมิดกฎเหล่านี้อาจนำไปสู่[บทลงโทษต่อบัญชีของคุณ](/wiki/Silence)

## Behaviour and conduct

- **เป้าหมายของ modding คือช่วยให้ mapper ปรับปรุงผลงานของตัวเอง** หากคุณไม่ได้สนใจช่วยคนอื่นหรือส่งเสริมการพูดคุยที่ดีต่อสุขภาพ ให้หลีกเลี่ยง discussion นั้น
- **คิดก่อนว่าคุณกำลังโพสต์อะไร ทำไมถึงโพสต์ และโพสต์ให้ใครอ่าน** สิ่งนี้จะช่วยให้เจตนาที่ดีของคุณถูกสื่อออกไป และทำให้โพสต์เข้าใจง่าย
- **เข้าถึงง่าย** Discussions เปิดให้ผู้ใช้ทุกคน ทุกคนยินดีเข้าร่วม รวมถึงคนที่เพิ่งเข้าคอมมูนิตี้ จงอดทน เข้าใจ และอธิบายเท่าที่จำเป็นเพื่อให้ทุกคนเรียนรู้และพัฒนาได้เร็วขึ้น
- **ปฏิบัติต่อผู้อื่นเหมือนที่คุณอยากให้คนอื่นปฏิบัติต่อคุณ** ผู้คนเปิดรับคำแนะนำที่ใจดีและมีประโยชน์มากกว่าคำพูดที่เป็นศัตรู อยู่ในประเด็นและหลีกเลี่ยงการโจมตีส่วนตัว คอมเมนต์เกลียดชัง และมุกที่ผิดที่ผิดทาง เพื่อไม่ให้ thread หลุดประเด็น
- **ใช้ [discussion page](/wiki/Beatmap_discussion)** พูดคุยข้อเสนอแนะของคุณกับสมาชิกคอมมูนิตี้คนอื่น ไม่อย่างนั้นข้อกังวลของคุณอาจไม่ถูกได้ยินโดยคนที่มีอำนาจจัดการอย่างเหมาะสม

## การตอบกลับ mod

เป็นความรับผิดชอบของ mapper ที่จะติดตาม discussions ล่าสุดในบีตแมปของตัวเอง การเมินข้อเสนอแนะหรือไม่ตอบกลับภายในเวลาที่เหมาะสมอาจทำให้ modders คนอื่นหมดกำลังใจที่จะร่วม discussion

เมื่อตอบข้อเสนอแนะของใครสักคน สิ่งสำคัญคือต้องพิจารณาสิ่งต่อไปนี้:

- **มองข้อเสนอแนะทั้งหมดว่ามีเหตุผลและสำคัญเท่าเทียมกัน** อย่าด่วนสรุปว่าข้อเสนอแนะ valid หรือไม่โดยไม่ได้ประเมินอย่างเหมาะสม
- **อธิบายตัวเองให้ชัดเจน** บอกว่าคุณเปลี่ยนอะไร และถ้าคุณไม่เห็นด้วยกับข้อเสนอแนะ ให้อธิบายว่าทำไมและทำอะไรแทน สิ่งนี้ไม่เพียงช่วยให้ modder ได้เรียนรู้ แต่ยังช่วยให้ mods ในอนาคตปรับให้เข้ากับ concept หรือ style ของคุณได้
- **ใจเย็นไว้** พูดสิ่งที่ต้องพูดเพื่ออธิบายตัวเองแล้วพอแค่นั้น ไม่ว่าข้อเสนอแนะจะไม่ช่วยแค่ไหน การตอบกลับด้วยความเป็นศัตรู ประชด ฯลฯ จะไม่ช่วยให้สถานการณ์ดีขึ้น
- **ให้ context** การให้ข้อมูลที่เกี่ยวข้องทั้งหมดในคำตอบต่อ mods จะทำให้ modders ในอนาคตเข้าใจ discussion ได้ง่ายขึ้น

## การแลกเปลี่ยนและ discussion ต่อ

บางครั้ง discussions อาจยืดยาว และ mod กับคำตอบเพียงชุดเดียวไม่พอให้ทุกฝ่ายตกลงกันได้ บางที mapper อาจไม่เห็นด้วยกับข้อเสนอแนะ และ modder ก็ไม่พอใจกับคำตอบของพวกเขา ตามอุดมคติ discussions ทั้งหมดควรหาข้อสรุปได้ภายในเวลาที่สมควร

พิจารณาแนวทางต่อไปนี้เมื่อไม่สามารถตกลงกันได้:

- **ถามหากคุณไม่แน่ใจอะไรบางอย่าง** ขอให้คนที่เกี่ยวข้องอธิบายว่าพวกเขาหมายถึงอะไร พวกเขาควรเต็มใจอธิบายให้คุณละเอียดขึ้น
- **ถามคนที่มีประสบการณ์มากกว่า เช่น สมาชิก [BN](/wiki/People/Beatmap_Nominators) หรือ [NAT](/wiki/People/Nomination_Assessment_Team)** แม้พวกเขาจะยุ่ง แต่ควรช่วยให้คุณหาข้อตกลงได้
- **เปิดใจ** หากคุณรู้ตัวว่ามีทางเลือกที่เหมาะกว่าสิ่งที่คุณเสนอไปตอนแรก อย่ากลัวที่จะวางศักดิ์ศรีลง ไม่มีความคิดเห็นที่ผิด ดังนั้นลองเข้าใจมุมมองอื่นแทนที่จะโจมตีมัน
- **หากสถานการณ์เริ่มเดือด ให้พักก่อน** สภาพจิตใจที่ถูกต้องสำคัญต่อการพูดคุย หากคุณอธิบายบางอย่างด้วยเหตุผลไม่ได้ คุณก็จะอธิบายด้วยความรุนแรงไม่ได้เช่นกัน
- **รายงานพฤติกรรมที่ไม่ดี** เมื่อเรื่องเริ่มดูคุมไม่อยู่ หรือ discussion เริ่มพยายามอยู่ในประเด็นไม่ไหว ให้[เรียก moderator](/wiki/Reporting_bad_behaviour) อย่าช่วยเพิ่มปัญหาด้วยการโพสต์ต่อ
