# .osu (รูปแบบไฟล์)

**`.osu`** คือรูปแบบไฟล์ข้อความที่มนุษย์สามารถอ่านได้ ซึ่งบรรจุข้อมูลเกี่ยวกับ [Beatmap](/wiki/Beatmap)

## โครงสร้าง (Structure)

บรรทัดแรกของไฟล์จะระบุเวอร์ชันของรูปแบบไฟล์ ตัวอย่างเช่น `osu file format v14` คือเวอร์ชันล่าสุด (สำหรับ [osu!(lazer)](/wiki/Client/Release_stream/Lazer) จะเป็นเวอร์ชัน v128)

เนื้อหาภายในไฟล์จะถูกแบ่งออกเป็นส่วนๆ (Sections) โดยมีชื่อส่วนระบุไว้ในวงเล็บเหลี่ยม

| ส่วน (Section) | คำอธิบาย | ประเภทเนื้อหา |
| :-- | :-- | :-- |
| `[General]` | ข้อมูลทั่วไปเกี่ยวกับ Beatmap | คู่ของ `คีย์: ค่า` |
| `[Editor]` | การตั้งค่าที่บันทึกไว้สำหรับตัวแก้ไข Beatmap | คู่ของ `คีย์: ค่า` |
| `[Metadata]` | [ข้อมูล](/wiki/Client/Beatmap_editor/Song_setup#general) ที่ใช้ระบุตัวตนของ Beatmap | คู่ของ `คีย์: ค่า` |
| `[Difficulty]` | [การตั้งค่าความยาก](/wiki/Client/Beatmap_editor/Song_setup#difficulty) | คู่ของ `คีย์: ค่า` |
| `[Events]` | เหตุการณ์กราฟิกของ Beatmap และ Storyboard | รายการที่คั่นด้วยเครื่องหมายจุลภาค |
| `[TimingPoints]` | จุดจังหวะและจุดควบคุม | รายการที่คั่นด้วยเครื่องหมายจุลภาค |
| `[Colours]` | สีคอมโบและสีของ Skin | คู่ของ `คีย์ : ค่า` |
| `[HitObjects]` | วัตถุที่ใช้เล่น (Hit objects) | รายการที่คั่นด้วยเครื่องหมายจุลภาค |

## Events (เหตุการณ์)

*ไวยากรณ์ของเหตุการณ์:* `eventType,startTime,eventParams`

- **`eventType` (String หรือ Integer):** ประเภทของเหตุการณ์ บางเหตุการณ์อาจระบุด้วยชื่อหรือตัวเลขก็ได้
- **`startTime` (Integer):** เวลาที่เริ่มเหตุการณ์ (หน่วยมิลลิวินาที) นับจากจุดเริ่มต้นของไฟล์เสียง สำหรับเหตุการณ์ที่ไม่มีเวลาเริ่ม ค่าเริ่มต้นจะเป็น `0`
- **`eventParams` (รายการคั่นด้วยจุลภาค):** พารามิเตอร์เพิ่มเติมตามประเภทของเหตุการณ์นั้นๆ

### Backgrounds (ภาพพื้นหลัง)

*ไวยากรณ์พื้นหลัง:* `0,0,filename,xOffset,yOffset`

- **`filename` (String):** ตำแหน่งของไฟล์ภาพพื้นหลังเมื่อเทียบกับไดเรกทอรีของ Beatmap โดยปกติจะล้อมรอบด้วยเครื่องหมายอัญประกาศ
- **`xOffset` (Integer)** และ **`yOffset` (Integer):** ออฟเซ็ตในหน่วย [osu! pixels](/wiki/Client/Beatmap_editor/osu!_pixel) จากจุดกึ่งกลางหน้าจอ เช่น ออฟเซ็ต `50,100` จะทำให้ภาพพื้นหลังเยื้องไปทางขวา 50 และลงด้านล่าง 100 หากเป็น `0,0` สามารถเลือกไม่เขียนก็ได้

### Videos (วิดีโอ)

*ไวยากรณ์วิดีโอ:* `Video,startTime,filename,xOffset,yOffset`

คำว่า `Video` สามารถแทนที่ด้วยเลข `1` ได้

- **`filename`**, **`xOffset`**, และ **`yOffset`** มีพฤติกรรมการทำงานเหมือนกับภาพพื้นหลัง

### Breaks (ช่วงพัก)

*ไวยากรณ์ช่วงพัก:* `2,startTime,endTime`

เลข `2` สามารถแทนที่ด้วยคำว่า `Break` ได้

- **`endTime` (Integer):** เวลาที่สิ้นสุดช่วงพัก (หน่วยมิลลิวินาที)

### Storyboards

*สำหรับข้อมูลเกี่ยวกับไวยากรณ์ Storyboard ดูที่ [Storyboard Scripting](/wiki/Storyboard/Scripting)*

Storyboard สามารถกำหนดแยกไว้ในไฟล์นามสกุล `.osb` ได้ ซึ่ง Storyboard ภายนอกนี้จะถูกใช้ร่วมกันในทุกระดับความยากของชุดแมพนั้น

อย่างไรก็ตาม แต่ละระดับความยากสามารถมี Storyboard เฉพาะของตนเองในไฟล์ `.osu` ได้เช่นกัน โดยจะใช้ควบคู่ไปกับ Storyboard ภายนอกหรือใช้เพียงอย่างเดียวก็ได้

## Timing points (จุดจังหวะ)

จุดจังหวะแต่ละจุดจะส่งผลต่อส่วนหนึ่งของแมพที่ระบุไว้ ซึ่งมักเรียกว่า "Timing section" รูปแบบไฟล์ `.osu` กำหนดให้จุดเหล่านี้ต้องเรียงลำดับตามเวลา

*ไวยากรณ์จุดจังหวะ:* `time,beatLength,meter,sampleSet,sampleIndex,volume,uninherited,effects`

- **`time` (Integer):** เวลาที่เริ่มส่วนของจังหวะ (หน่วยมิลลิวินาที)
- **`beatLength` (Decimal):** คุณสมบัตินี้มีความหมายสองอย่าง:
  - สำหรับจุดจังหวะแบบ Uninherited คือ ระยะเวลาของหนึ่งจังหวะ (หน่วยมิลลิวินาที)
  - สำหรับจุดจังหวะแบบ Inherited คือ ตัวคูณความเร็ว Slider แบบติดลบและเป็นเปอร์เซ็นต์ เช่น `-50` จะทำให้ Slider ในส่วนนี้เร็วเป็นสองเท่าของ `SliderMultiplier`
- **`meter` (Integer):** จำนวนจังหวะในหนึ่งห้องดนตรี (Measure) จุดจังหวะแบบ Inherited จะละเว้นค่านี้
- **`sampleSet` (Integer):** ชุดตัวอย่างเสียงพื้นฐาน (0 = ตามที่ตั้งไว้ในแมพ, 1 = normal, 2 = soft, 3 = drum)
- **`sampleIndex` (Integer):** ดัชนีของชุดเสียงที่กำหนดเอง ค่า `0` หมายถึงเสียงพื้นฐานของ osu!
- **`volume` (Integer):** ระดับความดังของเสียงเป็นเปอร์เซ็นต์
- **`uninherited` (0 หรือ 1):** กำหนดว่าเป็นจุดจังหวะแบบ Uninherited (1) หรือไม่
- **`effects` (Integer):** ค่าบิตแฟล็กสำหรับเอฟเฟกต์เพิ่มเติม ดูรายละเอียดใน [ส่วนของเอฟเฟกต์](#เอฟเฟกต์-(effects))

### เอฟเฟกต์ (Effects)

จุดจังหวะมีเอฟเฟกต์เพิ่มเติม 2 อย่างที่สามารถเปิดปิดได้โดยใช้บิตที่ 0 และ 3 ในเลขจำนวนเต็ม `effects`:

- 0: เปิดใช้งาน [Kiai time](/wiki/Gameplay/Kiai_time) หรือไม่
- 3: ซ่อนเส้นแบ่งห้องดนตรี (Bar line) ในโหมด osu!taiko และ osu!mania หรือไม่

บิตที่เหลือยังไม่มีการใช้งาน

### ตัวอย่าง

```
10000,333.33,4,0,0,100,1,1
12000,-25,4,3,0,100,0,1
```

จุดจังหวะแรกที่ 10 วินาที เป็นแบบ uninherited มีการตั้งค่าดังนี้:

- BPM เป็น 180 (`1 / 333.33 * 1000 * 60`)
- เครื่องหมายกำหนดจังหวะเป็น 4/4
- ชุดตัวอย่างเสียงตามที่ตั้งไว้ในแมพ
- ดัชนีตัวอย่างเสียงตามเสียงพื้นฐานของ osu!
- ความดัง 100%
- เปิดช่วง Kiai time

จุดจังหวะที่สองที่ 12 วินาที เป็นแบบ inherited มีการเปลี่ยนความเร็ว Slider เป็น 4 เท่า และเปลี่ยนชุดตัวอย่างเสียงเป็น Drum

## Colours (สีสัน)

ตัวเลือกทั้งหมดในส่วนนี้ใช้แทนสีต่างๆ ข้อมูลเป็นตัวเลข 3 ชุดคั่นด้วยจุลภาค (0–255) แทนค่าสีแดง (Red), เขียว (Green) และน้ำเงิน (Blue) ตามลำดับ

| ตัวเลือก | คำอธิบาย |
| :-- | :-- |
| `Combo#` (# คือตัวเลข) | สีคอมโบเพิ่มเติม |
| `SliderTrackOverride` | สีของเส้นทาง Slider |
| `SliderBorder` | สีขอบของ Slider |

## Hit objects (วัตถุที่ใช้เล่น)

*ไวยากรณ์วัตถุ:* `x,y,time,type,hitSound,objectParams,hitSample`

- **`x` (Integer)** และ **`y` (Integer):** ตำแหน่งพิกัดในหน่วย [osu! pixels](/wiki/Client/Beatmap_editor/osu!_pixel)
- **`time` (Integer):** เวลาที่ต้องกดวัตถุ (หน่วยมิลลิวินาที)
- **`type` (Integer):** ค่าบิตแฟล็กระบุประเภทของวัตถุ ดูรายละเอียดใน [ส่วนของประเภทวัตถุ](#ประเภท-(type))
- **`hitSound` (Integer):** ค่าบิตแฟล็กระบุเสียง Hitsound ที่ใช้ ดูรายละเอียดใน [ส่วนของเสียง Hitsound](#เสียง-hitsound-(hitsounds))
- **`objectParams` (รายการคั่นด้วยจุลภาค):** พารามิเตอร์เพิ่มเติมตามประเภทวัตถุ
- **`hitSample` (รายการคั่นด้วยโคลอน):** ข้อมูลชุดเสียงที่จะเล่นเมื่อกดวัตถุ สัมพันธ์กับ `hitSound` หากไม่ระบุ ค่าเริ่มต้นจะเป็น `0:0:0:0:`

### ประเภท (Type)

พารามิเตอร์ประเภทของวัตถุคือเลขจำนวนเต็ม 8 บิต โดยแต่ละบิตมีความหมายพิเศษดังนี้:

| ดัชนีบิต | ความหมาย |
| :-- | :-- |
| 0 | วัตถุเป็นวงกลม (Hit circle) |
| 1 | วัตถุเป็นสไลเดอร์ (Slider) |
| 2 | เป็นจุดเริ่มต้นของคอมโบใหม่ (New combo) |
| 3 | วัตถุเป็นสปินเนอร์ (Spinner) |
| 4, 5, 6 | เลขจำนวนเต็ม 3 บิตสำหรับระบุจำนวนสีคอมโบที่จะข้าม (Colour hax) จะมีผลเฉพาะเมื่อวัตถุนั้นเริ่มคอมโบใหม่ |
| 7 | วัตถุเป็นโน้ตยาวของ osu!mania (Hold note) |

### เสียง Hitsound (Hitsounds)

ค่าบิตแฟล็ก `hitSound` กำหนดเสียงที่จะเล่นเมื่อมีการกดวัตถุ:

| ดัชนีบิต | ความหมาย |
| :-- | :-- |
| 0 | Normal |
| 1 | Whistle |
| 2 | Finish |
| 3 | Clap |

หากไม่มีการตั้งค่าบิตใดๆ ระบบจะใช้เสียงพื้นฐาน (Normal)

สำหรับทุกโหมดเกมยกเว้น osu!mania คุณสมบัติ Skin `LayeredHitSounds` จะบังคับให้มีเสียง Normal เสมอไม่ว่าจะตั้งค่าบิตที่ 0 ไว้อย่างไร (เปิดใช้งานเป็นค่าเริ่มต้น)

#### ชุดเสียงที่กำหนดเอง (Custom hit samples)

การใช้ `hitSample` ช่วยให้ปรับแต่งเสียงได้ละเอียดขึ้น หากไม่ระบุจะมีค่าเป็น `0:0:0:0:`

*ไวยากรณ์:* `normalSet:additionSet:index:volume:filename`

- **`normalSet` (Integer):** ชุดเสียงสำหรับเสียง Normal
- **`additionSet` (Integer):** ชุดเสียงสำหรับเสียง Whistle, Finish และ Clap
- **`index` (Integer):** ลำดับของชุดเสียง หากเป็น `0` จะใช้ดัชนีตามจุดจังหวะปัจจุบัน
- **`volume` (Integer):** ความดังของเสียง (1-100) หากเป็น `0` จะใช้ความดังตามจุดจังหวะปัจจุบัน
- **`filename` (String):** ชื่อไฟล์เสียงที่ต้องการใช้เป็นพิเศษ

ค่า `normalSet` และ `additionSet` สามารถระบุได้ดังนี้:
- `0`: ไม่ระบุชุดเสียงพิเศษ (ใช้ตามจุดจังหวะ)
- `1`: ชุดเสียง Normal
- `2`: ชุดเสียง Soft
- `3`: ชุดเสียง Drum

ชื่อไฟล์เสียงที่ระบบจะเรียกใช้คือ `<sampleSet>-hit<hitSound><index>.wav` โดยระบบจะค้นหาตามลำดับจาก โฟลเดอร์ Beatmap -> โฟลเดอร์ Skin -> ทรัพยากรพื้นฐานของ osu!

หากมีการระบุ `filename` ระบบจะเล่นไฟล์นั้นจากโฟลเดอร์ Beatmap โดยตรงและไม่เล่นเสียง Addition อื่นๆ

### Hit circles (วงกลม)

วงกลมจะไม่มีข้อมูล `objectParams` เพิ่มเติม

### Sliders (สไลเดอร์)

*ไวยากรณ์สไลเดอร์:* `x,y,time,type,hitSound,curveType|curvePoints,slides,length,edgeSounds,edgeSets,hitSample`

- **`curveType` (Character):** ประเภทของเส้นโค้ง (`B` = bézier, `C` = centripetal catmull-rom, `L` = linear, `P` = perfect circle)
- **`curvePoints` (รายการพิกัดคั่นด้วย |):** จุดยึดที่ใช้สร้าง Slider ในรูปแบบ `x:y`
- **`slides` (Integer):** จำนวนครั้งที่ผู้เล่นต้องลากไปกลับ (เท่ากับจำนวน Repeat + 1)
- **`length` (Decimal):** ความยาวทางสายตาในหน่วย [osu! pixels](/wiki/Client/Beatmap_editor/osu!_pixel)
- **`edgeSounds` (รายการตัวเลขคั่นด้วย |):** เสียง Hitsound ที่เล่นตรงขอบ/จุดวกกลับของ Slider
- **`edgeSets` (รายการข้อความคั่นด้วย |):** ชุดตัวอย่างเสียงที่ใช้สำหรับแต่ละขอบ ในรูปแบบ `normalSet:additionSet`

#### เส้นโค้งของ Slider (Slider curves)

ในการสร้างเส้นโค้ง `x` และ `y` คือพิกัดของจุดแรก และ `curvePoints` คือจุดที่เหลือ

ประเภทเส้นโค้งใน osu!:
- **Bézier (B):** เส้นโค้งเบซีเยที่สามารถต่อกันได้หลายชุดโดยการวางจุดทับซ้อนกัน (จุดสีแดง)
- **Centripetal catmull-rom (C):** เส้นโค้งแบบ Catmull ปัจจุบันไม่ค่อยนิยมใช้เนื่องจากความสวยงามน้อยกว่า
- **Linear (L):** เส้นตรงระหว่างจุดควบคุมทั้งหมด
- **Perfect circle (P):** วงกลมที่สมบูรณ์แบบ กำหนดด้วยจุด 3 จุด (รวมจุดเริ่ม) หากมีมากกว่า 3 จุดจะกลายเป็นเส้นโค้งเบซีเยโดยอัตโนมัติ

หากค่า `length` ยาวกว่าความยาวของเส้นโค้งที่ระบุไว้ ปลาย Slider จะถูกลากต่อออกไปเป็นเส้นตรงจนครบความยาวที่กำหนด

#### เสียงของ Slider (Slider hitsounds)

นอกจากเสียงที่ขอบแล้ว Slider ยังมีเสียงที่เล่นต่อเนื่องในขณะที่ผู้เล่นลากตาม โดยจะเล่นวนซ้ำไปเรื่อยๆ จนกว่าจะจบ Slider รองรับเฉพาะเสียง Normal และ Whistle เท่านั้น ชื่อไฟล์คือ `<sampleSet>-slider<hitSound><index>.wav`

### Spinners (สปินเนอร์)

*ไวยากรณ์สปินเนอร์:* `x,y,time,type,hitSound,endTime,hitSample`

- **`endTime` (Integer):** เวลาที่สิ้นสุดสปินเนอร์ (หน่วยมิลลิวินาที)
- พิกัด `x` และ `y` ไม่มีผลต่อสปินเนอร์ โดยค่าเริ่มต้นจะอยู่ที่กึ่งกลางสนามเล่น `256,192`

### Holds (โน้ตยาว - เฉพาะ osu!mania)

*ไวยากรณ์โน้ตยาว:* `x,y,time,type,hitSound,endTime:hitSample`

- **`endTime` (Integer):** เวลาที่สิ้นสุดโน้ตยาว (หน่วยมิลลิวินาที)
- `x` ใช้กำหนดลำดับของแถว (Column) คำนวณจาก `floor(x * columnCount / 512)`
- `y` ไม่มีผลต่อโน้ตยาว โดยค่าเริ่มต้นจะอยู่ที่กึ่งกลางสนามเล่น `192`

### ตัวอย่างการแปลงโหมด

#### osu!taiko
ใช้เพียงข้อมูล `time` เพื่อวางตำแหน่งโน้ต พิกัด `x` และ `y` จะถูกละเว้น ส่วนความโค้งของ Slider จะสนใจเพียง `length` เท่านั้น สีคอมโบไม่มีผล และมีการใช้ Hitsound เฉพาะโหมด (Whistle/Clap เป็นสีน้ำเงิน, Finish เป็นโน้ตใหญ่)

#### osu!catch
ใช้พิกัดแกน x เท่านั้น พิกัดแกน y จะถูกละเว้น เส้นโค้งของ Slider อาจใช้พื้นที่แนวตั้งเพื่อสร้างการเร่งความเร็วในแนวนอนได้

#### osu!mania
ใช้พิกัดแกน x เพื่อกำหนดแถว พิกัดแกน y จะถูกละเว้น ไม่มีการใช้ Slider และ Spinner
