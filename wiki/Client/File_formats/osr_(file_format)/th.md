# .osr (รูปแบบไฟล์)

**.osr** คือรูปแบบไฟล์สำหรับอธิบายข้อมูล [Replay (บันทึกการเล่น)](/wiki/Gameplay/Replay) ของ osu! ไฟล์นามสกุล `.osr` จะสามารถเปิดเล่นได้ก็ต่อเมื่อตัวเกมมีไฟล์ [Beatmap](/wiki/Beatmap) ชุดเดียวกับที่ใช้ในการบันทึก Replay นั้นอยู่เท่านั้น

## ประเภทข้อมูล (Data Types)

| ชื่อ | ขนาด (ไบต์) | คำอธิบาย |
| :-- | :-- | :-- |
| Byte | 1 | ค่าตัวเลข 8 บิตตัวเดียว |
| Short | 2 | ค่า Little endian ขนาด 2 ไบต์ |
| Integer | 4 | ค่า Little endian ขนาด 4 ไบต์ |
| Long | 8 | ค่า Little endian ขนาด 8 ไบต์ |
| ULEB128 | ไม่คงที่ | เลขจำนวนเต็มแบบความยาวไม่คงที่ ดูรายละเอียดที่ [ULEB128](https://en.wikipedia.org/wiki/LEB128) |
| String | ไม่คงที่ | ประกอบด้วย 3 ส่วน: ไบต์เดียวที่มีค่า 0x00 (ไม่มีข้อความถัดไป) หรือ 0x0b (มีข้อความ), ตามด้วย ULEB128 ระบุความยาวของข้อความ และสุดท้ายคือตัวข้อความที่เข้ารหัสแบบ UTF-8 ดูรายละเอียดที่ [UTF-8](https://en.wikipedia.org/wiki/UTF-8) |

## โครงสร้างไฟล์ (Format)

ตารางนี้ไม่ได้ระบุพิกัดออฟเซ็ตของไบต์ เนื่องจากข้อมูลบางส่วนมีความยาวไม่คงที่

| ประเภทข้อมูล | คำอธิบาย |
| :-- | :-- |
| Byte | โหมดเกมของ Replay (0 = osu!, 1 = osu!taiko, 2 = osu!catch, 3 = osu!mania) |
| Integer | เวอร์ชันของตัวเกมขณะที่สร้าง Replay (เช่น 20131216) |
| String | MD5 hash ของ Beatmap |
| String | ชื่อผู้เล่น |
| String | MD5 hash ของ Replay (รวมถึงคุณสมบัติบางอย่างของ Replay นั้นๆ) |
| Short | จำนวนคะแนน 300 |
| Short | จำนวนคะแนน 100 ในโหมด osu!, 150 ใน osu!taiko, 100 ใน osu!catch, 100 ใน osu!mania |
| Short | จำนวนคะแนน 50 ในโหมด osu!, ผลไม้เล็กใน osu!catch, 50 ใน osu!mania |
| Short | จำนวนคะแนน Geki ในโหมด osu!, Max 300 ใน osu!mania |
| Short | จำนวนคะแนน Katu ในโหมด osu!, 200 ใน osu!mania |
| Short | จำนวนคะแนน Miss |
| Integer | คะแนนรวมทั้งหมดที่แสดงในหน้าสรุปผล |
| Short | คอมโบสูงสุดที่แสดงในหน้าสรุปผล |
| Byte | สถานะ Perfect/Full Combo (1 = ไม่พลาดเลย, ไม่หลุดสไลเดอร์ และลากสไลเดอร์จนจบ) |
| Integer | Mod ที่ใช้งาน ดูรายการค่าของ Mod ได้ที่ด้านล่าง |
| String | กราฟเส้นพลังชีวิต: คู่ตัวเลข u/v คั่นด้วยเครื่องหมายจุลภาค โดย u คือตำแหน่งเวลาในหน่วยมิลลิวินาที และ v คือค่าทศนิยม 0 - 1 ที่แสดงปริมาณพลังชีวิตในเวลานั้น (0 = เลือดหมด, 1 = เลือดเต็ม) |
| Long | ข้อมูลเวลา (Timestamp) ในรูปแบบ [Windows ticks](https://learn.microsoft.com/en-us/dotnet/api/system.datetime.ticks) |
| Integer | ความยาวของข้อมูล Replay ที่ถูกบีบอัด (หน่วยไบต์) |
| Byte Array | ข้อมูล Replay ที่ถูกบีบอัด |
| Long | รหัสคะแนนออนไลน์ (Online Score ID) |
| Double | ข้อมูล Mod เพิ่มเติม จะปรากฏเฉพาะเมื่อเปิดใช้งาน [Target Practice](/wiki/Gameplay/Game_modifier/Target_Practice) |

**ข้อมูล Mod เพิ่มเติม:**

| Mod | ข้อมูลที่เก็บไว้ |
| :-- | :-- |
| Target Practice | ความแม่นยำรวมจากการกดทั้งหมด ให้นำค่านี้ไปหารด้วยจำนวนเป้าหมายในแมพเพื่อหาค่าความแม่นยำที่แสดงในเกม |

ข้อมูลส่วนที่เหลือประกอบด้วยรายละเอียดการเคลื่อนที่ของเมาส์และการกดปุ่มในรูปแบบสตรีม [LZMA](https://en.wikipedia.org/wiki/Lempel–Ziv–Markov_chain_algorithm)

เมื่อแตกข้อมูลออกมา ข้อความจะประกอบด้วยชุดข้อมูลที่คั่นด้วยเครื่องหมายจุลภาค แต่ละชุดคือหนึ่งการกระทำ แสดงด้วยตัวเลข 4 ตัวในรูปแบบ: `w | x | y | z`

| ส่วน | ประเภทข้อมูล | คำอธิบาย |
| :-- | :-- | :-- |
| w | Long | ระยะเวลาในหน่วยมิลลิวินาทีนับจากการกระทำก่อนหน้า |
| x | Float | พิกัดแกน x ของเคอร์เซอร์ ตั้งแต่ 0 - 512 |
| y | Float | พิกัดแกน y ของเคอร์เซอร์ ตั้งแต่ 0 - 384 |
| z | Integer | การรวมกันของค่าบิตของปุ่มกด/เมาส์ (M1 = 1, M2 = 2, K1 = 4, K2 = 8, Smoke = 16) (โดยปกติ K1 จะถูกใช้คู่กับ M1 และ K2 คู่กับ M2 เสมอ เช่น 1+4=5; 2+8=10) |

สำหรับ Replay ที่สร้างในเวอร์ชัน `20130319` หรือใหม่กว่า จะมีการเก็บค่าสุ่ม (RNG seed) ขนาด 32 บิตที่ใช้ในการคำนวณคะแนนไว้ในเฟรมสุดท้ายของสตรีม LZMA ในรูปแบบ `-12345|0|0|seed`
