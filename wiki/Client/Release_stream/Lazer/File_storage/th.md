# การจัดเก็บไฟล์ใน osu!(lazer) (File storage in osu!(lazer))

โดยค่าเริ่มต้น osu!(lazer) จะจัดเก็บ Beatmap, Skin และไฟล์บันทึกการเล่น (Replay) ไว้ในไดเรกทอรีต่อไปนี้:

- `%appdata%/osu` (Windows)
- `~/.local/share/osu` (Linux)
- `~/Library/Application Support/osu` (macOS)
- `Android/data/sh.ppy.osulazer/files` (Android)

สำหรับแพลตฟอร์มคอมพิวเตอร์ คุณสามารถย้ายที่เก็บไฟล์ทั้งหมดไปยังตำแหน่งอื่นได้ โดยการใช้ปุ่ม `Change folder location...` ในเมนูการตั้งค่าของตัวเกม

## โครงสร้างการจัดเก็บ (Storage structure)

ใน osu!(stable) โครงสร้างการจัดเก็บข้อมูลแบบเดิมที่ผู้ใช้สามารถเข้าถึงไฟล์ได้โดยตรง มักจะก่อให้เกิดปัญหาตามมามากมายจากการที่ไฟล์สำคัญถูกดัดแปลงโดยไม่ตั้งใจ เพื่อหลีกเลี่ยงปัญหานี้ osu!(lazer) จึงเลือกใช้วิธีการจัดเก็บไฟล์ที่เข้มงวดมากขึ้น ด้วยเหตุนี้ จึงไม่มีโฟลเดอร์ที่เทียบเท่ากับ `Songs` และ `Skins` แบบเดิมอีกต่อไป โดยไฟล์ทั้งหมดที่นำเข้าสู่ lazer จะถูกเปลี่ยนชื่อตามค่า [SHA-256 hashes](https://en.wikipedia.org/wiki/SHA-2) และข้อมูลความเชื่อมโยงของไฟล์เหล่านี้จะถูกเก็บไว้ในฐานข้อมูลของตัวเกม

ตัวอย่างเช่น ไฟล์ที่มีค่า SHA-256 hash เป็น

```
1a47929b6056d34d25a95eeb2012395ceed66af6f40cc37c898a08482d6325d2
```

จะถูกจัดเก็บไว้ในตำแหน่ง

```
files/1/1a/1a47929b6056d34d25a95eeb2012395ceed66af6f40cc37c898a08482d6325d2
```

วิธีนี้ช่วยประหยัดพื้นที่จัดเก็บข้อมูลโดยการป้องกันไม่ให้มีไฟล์ที่ซ้ำกันถูกบันทึกลงในดิสก์ และยังป้องกันไม่ให้ผู้ใช้ (หรือโปรแกรมอื่น) เข้ามาดัดแปลงไฟล์ที่สำคัญได้โดยง่าย ความไม่สะดวกต่างๆ ที่เกิดขึ้นเมื่อเทียบกับเวอร์ชัน Stable ได้รับการแก้ไขแล้วด้วยการอนุญาตให้ส่งออกไฟล์ (Export) หรือการเพิ่มฟีเจอร์จัดการภายในเกมแบบใหม่ และจะมีการพัฒนาฟีเจอร์อื่นๆ เพิ่มเติมในอนาคต

ในขณะนี้ หากคุณต้องการแก้ไข Skin หรือ Beatmap ด้วยตนเองนอกเหนือจากเครื่องมือที่มีให้ในเกม วิธีที่ถูกต้องคือการส่งออก (Export) รายการนั้นออกมา แก้ไขให้เรียบร้อย แล้วจึงนำเข้า (Import) กลับเข้าไปใหม่อีกครั้งพร้อมกับการเปลี่ยนแปลงที่ต้องการ

## การย้ายข้อมูลจาก osu!(stable) (Migration)

### ผ่านระบบ Hard links

ในระบบส่วนใหญ่ osu!(lazer) จะสามารถนำเข้าข้อมูลจากเวอร์ชัน Stable ได้โดยไม่ต้องสร้างสำเนาข้อมูลชุดที่สองลงในดิสก์ สิ่งนี้ทำได้ด้วยคุณสมบัติของระบบปฏิบัติการที่เรียกว่า *Hard links*

Hard link มีแนวคิดคล้ายกับ *Shortcut* (ทางลัด) คือการทำให้ไฟล์เดียวกันปรากฏอยู่ในหลายๆ ที่บนระบบไฟล์ของคุณ อย่างไรก็ตาม ในขณะที่ Shortcut เป็นเพียงไฟล์ธรรมดาที่ชี้ไปยังอีกไฟล์หนึ่ง (และต้องใช้พื้นที่เพิ่มเติม) Hard link จะทำงานในระดับที่ลึกกว่านั้น คือในระดับของตัวระบบไฟล์เอง

พูดง่ายๆ คือ ไฟล์สองไฟล์ที่ทำ Hard link กัน คือชื่อเรียกสองชื่อที่ชี้ไปยัง "พื้นที่กายภาพเดียวกัน" บนฮาร์ดไดรฟ์ ซึ่งหมายความว่าเมื่อการย้ายข้อมูลเสร็จสิ้น ทั้งโฟลเดอร์ `Songs` ในเวอร์ชัน Stable และโฟลเดอร์ `files` ในเวอร์ชัน lazer จะประกอบด้วยไฟล์ที่ชี้ไปยังข้อมูลชุดเดียวกันบนดิสก์ การลบไฟล์จากที่หนึ่งจะไม่มีผลกับอีกที่หนึ่ง และการอัปเดตหรือแก้ไขไฟล์จะทำให้ไฟล์เหล่านั้นแยกออกจากกันโดยอัตโนมัติ ซึ่งจะไม่ส่งผลกระทบต่ออีกเวอร์ชันตราบใดที่คุณใช้ตัวเกม osu! ในการแก้ไข

โปรดทราบว่าหากคุณตรวจสอบการใช้พื้นที่ดิสก์ผ่านหน้าต่าง `Properties` ใน Windows Explorer หรือวิธีที่คล้ายกัน จะดูเหมือนว่าทั้งโฟลเดอร์ `Songs` และโฟลเดอร์ `files` ต่างก็ใช้พื้นที่ดิสก์ปริมาณมาก ซึ่งอาจทำให้ผู้ใช้เข้าใจผิดว่าข้อมูลถูกเก็บไว้ซ้ำซ้อนกันสองชุด สิ่งนี้เป็นเพียงผลจากการคำนวณขนาดไฟล์ของระบบ แต่ข้อมูลจริงๆ **ไม่ได้** ถูกทำซ้ำ คุณสามารถตรวจสอบเรื่องนี้ได้โดยการดูพื้นที่ว่างทั้งหมดของฮาร์ดไดรฟ์ก่อนและหลังการย้ายข้อมูล

#### ระบบที่รองรับ

เนื่องจาก Hard link เป็นฟีเจอร์ระดับระบบไฟล์ ไฟล์ที่ทำ Hard link กันจะต้องอยู่ในไดรฟ์เดียวกันเท่านั้น

- **Windows**: ไดรฟ์ต้องถูกฟอร์แมตเป็น NTFS
- **macOS**: ระบบไฟล์ต้องรองรับ Hard links
- **Linux**: ระบบไฟล์ต้องรองรับ Hard links

### ผ่านการคัดลอกไฟล์ (File copy)

สำหรับระบบปฏิบัติการหรือระบบไฟล์ที่ไม่รองรับ Hard links (หรือฟังก์ชันดังกล่าวยังไม่ถูกพัฒนา) การย้ายข้อมูลจาก Stable ไปยัง lazer จะใช้วิธีการคัดลอกไฟล์ทั้งหมดจากตำแหน่งเดิมไปยังตำแหน่งใหม่ ซึ่งจะทำให้ข้อมูลทั้งสองเวอร์ชันแยกจากกันอย่างเด็ดขาด แต่จะใช้พื้นที่ดิสก์เพิ่มขึ้นสูงสุดถึงสองเท่า
