# ระบบการตัดสินของ osu! (osu! judgement system)

## การตัดสิน (Judgements)

**การตัดสิน (Judgement)** หรือ **ผลการกด (Hit result)** คือผลลัพธ์ที่ได้จากการโต้ตอบกับ [วัตถุ (Hit object)](/wiki/Gameplay/Hit_object) ภายในช่วงเวลาการกดที่กำหนด (Hit window) คะแนนและความแม่นยำจะถูกคำนวณตามประเภทของการตัดสินที่ได้รับ

โดยปกติผู้เล่นมักจะเรียกชื่อการตัดสินตามค่าคะแนนที่ได้รับ (ยกเว้นการกดพลาด) เช่น GREAT มักจะเรียกว่า "300" เป็นต้น

| รูปภาพ | ชื่อเรียก | [คะแนนที่ได้รับ](/wiki/Gameplay/Score/ScoreV1/osu!) | [ความแม่นยำ](/wiki/Gameplay/Accuracy#osu!) | ช่วงเวลาการกดสูงสุด (มิลลิวินาที) |
| :-: | :-: | --: | --: | :-- |
| ![](/wiki/shared/judgement/osu!/hit300.png) | GREAT | 300 | 100% | `80 - 6 × OD` |
| ![](/wiki/shared/judgement/osu!/hit100.png) | OK | 100 | 33.33% | `140 - 8 × OD` |
| ![](/wiki/shared/judgement/osu!/hit50.png) | MEH | 50 | 16.67% | `200 - 10 × OD` |
| ![](/wiki/shared/judgement/osu!/hit0.png) | MISS | 0 | 0% | `400` |

ช่วงเวลาการกดจะขึ้นอยู่กับค่า [ความยากโดยรวม (OD)](/wiki/Beatmap/Overall_difficulty) ของบีทแมพ การกดจะถือว่าอยู่ในช่วงเวลานั้นๆ หาก `ความคลาดเคลื่อน < ช่วงเวลาการกดสูงสุด` ซึ่งหมายความว่าค่าที่ระบุในตารางคือครึ่งหนึ่งของความกว้างของช่วงเวลาการกดทั้งหมด

เนื่องจากระบบมีการปัดเศษทศนิยมและตัดเศษจำนวนเต็ม ช่วงเวลาจริงอาจจะสั้นกว่าที่คำนวณจากสูตรได้สูงสุดถึง 1.5 มิลลิวินาทีในทั้งสองฝั่ง

### Geki และ Katu

วัตถุตัวสุดท้ายใน [ชุดคอมโบ (Combo set)](/wiki/Beatmapping/Combo) อาจจะให้ผลการตัดสินในรูปแบบพิเศษ โดยขึ้นอยู่กับผลการกดที่ทำได้ในชุดคอมโบนั้นๆ ผลการตัดสินเหล่านี้จะช่วยเพิ่ม [พลังชีวิต](/wiki/Gameplay/Health) ได้มากกว่าปกติ

| รูปภาพ | ชื่อเรียก | เป็นรูปแบบของ | เงื่อนไข |
| :-: | :-: | :-: | :-- |
| ![](/wiki/shared/judgement/osu!/hit300g.png) | [Geki](/wiki/Gameplay/Judgement/Geki) (激) | GREAT | ได้ระดับ GREAT ทุกวัตถุในชุดคอมโบ |
| ![](/wiki/shared/judgement/osu!/hit300k.png) | [Katu](/wiki/Gameplay/Judgement/Katu) (喝) | GREAT | ได้อย่างน้อยระดับ OK ทุกวัตถุในชุดคอมโบ และตัวสุดท้ายได้ GREAT |
| ![](/wiki/shared/judgement/osu!/hit100k.png) | [Katu](/wiki/Gameplay/Judgement/Katu) (喝) | OK | ได้อย่างน้อยระดับ OK ทุกวัตถุในชุดคอมโบ |

ระบบนี้อ้างอิงมาจากเกม [Osu! Tatakae! Ouendan](https://en.wikipedia.org/wiki/Osu!_Tatakae!_Ouendan) และ [Elite Beat Agents](https://en.wikipedia.org/wiki/Elite_Beat_Agents) โดยในเวอร์ชันภาษาอังกฤษจะใช้คำว่า Elite Beat! และ Beat! แทนคำว่า Geki และ Katsu

## กลไกการตัดสิน (Judgement mechanics)

### Hit circles (วงกลม)

วงกลมจะถูกตัดสินเป็น GREAT, OK, MEH หรือ MISS ตามความแม่นยำของจังหวะที่กด การกดวงกลมก่อนที่จะถึงช่วงเวลา MISS จะไม่มีผลใดๆ (นอกจากทำให้เกิด [notelock](/wiki/Gameplay/Judgement/Notelock)) และหากไม่กดวงกลมเลยจนเลยช่วงเวลา MEH ไปแล้ว จะถือว่าเป็น MISS ทันที

### Sliders (สไลเดอร์)

[สไลเดอร์](/wiki/Gameplay/Hit_object/Slider) ประกอบด้วยหลายส่วน ได้แก่: [หัวสไลเดอร์ (Slider head)](/wiki/Gameplay/Hit_object/Slider/Sliderhead), [หางสไลเดอร์ (Slider tail)](/wiki/Gameplay/Hit_object/Slider/Slidertail), [จุดจังหวะ (Slider ticks)](/wiki/Gameplay/Hit_object/Slider/Slider_tick) และ [ลูกศรย้อนกลับ (Slider repeats)](/wiki/Gameplay/Hit_object/Slider/Reverse_slider) การตัดสินสไลเดอร์ในภาพรวมจะอิงตามจำนวนส่วนประกอบที่ผู้เล่นเก็บได้ ดังนี้:

| การตัดสิน | การเก็บส่วนประกอบสไลเดอร์ |
| :-: | :-- |
| GREAT | เก็บได้ครบ 100% |
| OK | เก็บได้ 50% ขึ้นไป |
| MEH | เก็บได้เพียงส่วนเดียวขึ้นไป |
| MISS | ไม่ได้เก็บเลย 0% |

สำหรับการกดหัวสไลเดอร์ให้สำเร็จนั้น เพียงแค่กดให้อยู่ในช่วงเวลา MEH ก็เพียงพอแล้ว อย่างไรก็ตาม หากเปิดใช้งานระบบ [ScoreV2](/wiki/Gameplay/Game_modifier/ScoreV2) ความแม่นยำในการกดหัวสไลเดอร์จะถูกนำมาคิดด้วย โดยผลการตัดสินสูงสุดของสไลเดอร์นั้นจะถูกจำกัดไว้ไม่เกินระดับการตัดสินที่ได้รับจากการกดหัวสไลเดอร์

รายละเอียดเพิ่มเติมเกี่ยวกับผลกระทบต่อ [คอมโบ](/wiki/Gameplay/Combo_(score_multiplier)):

- การกดหัวสไลเดอร์เร็วเกินไป (ก่อนเข้าช่วงเวลา MEH), การเก็บจุดจังหวะพลาด หรือการพลาดลูกศรย้อนกลับ จะไม่ทำให้ได้รับ MISS แต่จะทำให้ [คอมโบหลุด (Combo break)](/wiki/Gameplay/Judgement/Combobreak) อย่างไรก็ตาม หากยังกดปุ่มค้างไว้ คุณยังสามารถเก็บส่วนประกอบที่เหลือของสไลเดอร์นั้นได้อยู่ เหตุการณ์นี้มักถูกเรียกว่า [Slider break](/wiki/Gameplay/Judgement/Slider_break)
- การพลาดหางสไลเดอร์จะไม่นับเป็น MISS แต่จะไม่เพิ่มจำนวนคอมโบให้

### Spinners (สปินเนอร์)

[สปินเนอร์](/wiki/Gameplay/Hit_object/Spinner) แต่ละตัวจะมีจำนวนรอบที่ต้องหมุนให้ครบเพื่อทำภารกิจให้สำเร็จ ซึ่งจำนวนรอบจะขึ้นอยู่กับค่า [ความยากโดยรวม (OD)](/wiki/Beatmap/Overall_difficulty#sliders-and-spinners) ความเร็วในการหมุนจะคำนวณจากความเร็วของเคอร์เซอร์ และไม่จำเป็นต้องตรงกับจำนวนรอบที่เคอร์เซอร์วิ่งรอบจุดศูนย์กลางเสมอไป

| การตัดสิน | จำนวนรอบที่ต้องการ[^half-spins] |
| :-: | :-- |
| GREAT[^spinner-clear] | 100% |
| OK | ขาดไปเพียงหนึ่งรอบจากจำนวนที่ต้องการ |
| MEH | ทำได้ 25% ของจำนวนที่ต้องการ |
| MISS | ทำได้ 0% |

สูตรการคำนวณจำนวนรอบที่ต้องการ:

|  |  |
| :-- | :-- |
| ความเร็วการหมุนขั้นต่ำต่อวินาที[^minimum-sps] | `1.5 + 0.2 × OD` (หาก OD < 5), `1.25 + 0.25 × OD` (หาก OD ≥ 5) |
| จำนวนรอบขั้นต่ำที่ต้องหมุน | (ความยาวสปินเนอร์เป็นวินาที × ความเร็วการหมุนขั้นต่ำต่อวินาที) + 0.5 |

หากสปินเนอร์สั้นมาก จำนวนรอบที่ต้องการอาจถูกคำนวณเป็น 0 ซึ่งจะทำให้สปินเนอร์นั้นสำเร็จด้วยระดับ GREAT เสมอโดยอัตโนมัติ

## ประวัติความเป็นมา (History)

อัลกอริทึมการตัดสินสปินเนอร์มีการเปลี่ยนแปลงครั้งใหญ่ในเวอร์ชัน [20190513.2 Stable](https://osu.ppy.sh/home/changelog/stable40/20190513.2) โดยมีความแตกต่างดังนี้:

- ความต่างระหว่างระดับ OK และ MEH รวมถึง MEH และ GREAT เดิมเท่ากับครึ่งรอบ ทำให้ยากต่อการได้รับผลการตัดสินที่ไม่ใช่ MISS มากกว่าเดิม
- สปินเนอร์ที่สั้นเกินไปในสมัยก่อนอาจทำให้ไม่สามารถหมุนให้สำเร็จได้เลย
- สปินเนอร์ทุกลูกเดิมต้องการการหมุนเพิ่มขึ้นอีกครึ่งรอบเพื่อทำให้สำเร็จ

คะแนนการเล่นที่ทำไว้ก่อนวันที่ 10 พฤษภาคม 2019 จะยังคงใช้อัลกอริทึมแบบเก่าในการประมวลผล Replay

## หมายเหตุ (Notes)

[^half-spins]: ภายในระบบจะคำนวณการหมุนเป็นแบบครึ่งรอบ (Half revolutions) สูตรที่แสดงในหน้านี้ถูกปรับให้เป็นจำนวนรอบเต็มเพื่อความเข้าใจง่าย โดยค่านจะถูกปัดเศษลงให้ใกล้เคียงครึ่งรอบมากที่สุด
[^spinner-clear]: ข้อความ ["Clear"](/wiki/Skinning/osu!#spinner) (`spinner-clear.png`) จะปรากฏขึ้นก่อนที่จะถึงจำนวนรอบที่ต้องการสำหรับระดับ GREAT ประมาณครึ่งรอบ ซึ่งอาจเกิดจากการตกหล่นในขั้นตอนการพัฒนา
[^minimum-sps]: เนื่องจากมีค่าคงที่ +0.5 ในสูตรจำนวนรอบขั้นต่ำ ความเร็วเฉลี่ยจริงที่ต้องการจะเร็วกว่าปกติประมาณ `0.5 / ความยาวสปินเนอร์เป็นวินาที` รอบต่อวินาที
