# คำถามที่พบบ่อยเกี่ยวกับ skinning

บทความนี้รวมคำถามเกี่ยวกับ skinning ที่มีคนถามกันบ่อย

## เริ่มต้นใช้งาน

### Skinning คืออะไร?

Skinning เป็นระบบเปลี่ยนรูปภาพแบบง่าย ๆ ที่ osu! จะโหลดขึ้นมาเมื่อเลือกสกินในตัวเลือก การประกอบสกินเข้าด้วยกันทำได้ไม่ยาก แต่การสร้าง asset สำหรับสกินนั้นยากกว่าเล็กน้อย

### มีใครทำสกินจากอนิเมะ/เกมเรื่องนั้นไหม?

ถ้าคุณไล่ดูทั้งฟอรัม skinning แล้วและมั่นใจว่าหา *สกินนั้น* ไม่เจอ ก็แปลว่ายินดีด้วย osu! ยังไม่มีสกินนั้น ถ้าอย่างนั้นคุณก็เริ่มลงมือสร้างสกินในฝันของตัวเอง แล้วทำให้มันเป็นสิ่งที่คนอื่นอาจอยากใช้ด้วยได้เลย!

**ถึงอย่างนั้นก็ตาม กรุณาอย่าไปขอให้คนอื่นทำสกินให้ในฟอรัมเด็ดขาด** ถ้าไม่ทำตามกฎนี้ กระทู้ของคุณจะถูกย้ายไปที่ wastelands

### จะหาโหลดสกินที่เห็นคนอื่นใช้เล่นได้จากที่ไหน?

อย่างแรก อย่าถามเรื่องนี้ในฟอรัม ไม่อย่างนั้นกระทู้ของคุณมีโอกาสสูงที่จะถูกโยนไปที่ wastelands อย่างที่สอง ทางที่ดีกว่าคือไปถามเจ้าของสกินคนนั้นโดยตรง เขาอาจให้สกินกับคุณก็ได้ แต่ก็ไม่ได้หมายความว่าจะได้เสมอไป

### เอา element จากสกินของคนอื่นมาใช้ได้ไหม?

ได้ คุณสามารถใช้ skinning element จากสกินอื่นได้ สกินแบบนี้เรียกว่า "mixed skins" หรือ "skin mashups" อย่างไรก็ตาม **คุณไม่ได้รับอนุญาตให้โพสต์สกินเหล่านี้ใน subforum skinning ไม่ว่าจะได้รับอนุญาตมาหรือไม่ก็ตาม!** แต่คุณยังสามารถโพสต์ไว้ในลายเซ็นหรือหน้าโปรไฟล์ me! ของตัวเองได้

ข้อยกเว้นเดียวคือเสียง เพราะโดยทั่วไปทำได้ยากกว่า จึงสามารถนำเสียงจากสกินอื่นมาใช้ได้ แม้ไม่ได้รับอนุญาตแบบชัดเจนจาก skinner ต้นฉบับ ถึงอย่างนั้น การให้เครดิต skinner ต้นฉบับก็เป็นเรื่องที่ควรทำ

### โฟลเดอร์สกินของฉันควรมีอะไรบ้าง?

ระหว่างที่กำลังพัฒนาสกิน คุณสามารถใส่อะไรก็ได้ไว้ในโฟลเดอร์สกิน รวมถึงโฟลเดอร์อื่น ๆ ด้วย osu! จะมองข้ามไฟล์เหล่านั้นไปเอง ดังนั้นคุณจะเก็บไฟล์ `.xcf` หรือ `.psd` ไว้ในโฟลเดอร์เดียวกันเพื่อกลับมาแก้ทีหลังก็ได้

แต่ตอนปล่อยสกิน คุณควรลบไฟล์เหล่านี้ออกเพื่อลดขนาดไฟล์ ถ้าคุณใช้ template skin อยู่ ให้ลบ element ที่ไม่ได้แก้ออก เพราะ osu! จะจัดการ element ที่หายไปด้วยค่าเริ่มต้นให้เอง

อย่างไรก็ตาม โปรดจำไว้ว่าโฟลเดอร์ชื่อ `taiko` (ไม่สนตัวพิมพ์ใหญ่เล็ก) ถูกสงวนไว้สำหรับสกิน osu!taiko ซึ่งจะ override element ของ osu!taiko ทั้งหมด (รวมถึง element อินเทอร์เฟซบางส่วนตอนเล่นบีตแมป osu!taiko)

### รีโหลดสกินใน osu! ทำอย่างไร? {#how-do-i-reload-my-skin-in-osu}

คุณต้องเลือกสกินนั้นอยู่ก่อน จากนั้นกดคีย์ลัด `Ctrl` + `Alt` + `Shift` + `S` เพื่อรีโหลดสกินใน osu!

คุณยังสามารถปิดแล้วเปิด osu! ใหม่ หรือเปิด updater เพื่อรีโหลดสกินได้เหมือนกัน แต่โดยปกติการใช้คีย์ลัดสี่ปุ่มนี้จะเร็วกว่าเยอะ

### Skinning sets คืออะไร?

*หน้าหลัก: [Ranking Criteria/Skin Set List](/wiki/Ranking_criteria/Skin_set_list)*

ใน osu! มี element ที่สามารถทำสกินได้มากกว่า 200 รายการ (ยังไม่นับเฟรมแยกของ animation) แต่คุณไม่จำเป็นต้องทำสกินให้ครบทุกอัน คุณสามารถลบ element ที่ไม่ต้องการออกได้ เพราะ osu! จะใช้รูปเริ่มต้นแทนส่วนที่คุณไม่ได้ใส่มา

อย่างไรก็ตาม skinning set (ตามที่กำหนดไว้ใน Ranking Criteria) จะบอกว่า ถ้ามี element ใด element หนึ่งใน set ถูกทำสกินแล้ว element อื่น ๆ ใน set เดียวกันต้องถูกทำสกินด้วย ตัวอย่างเช่น ผู้เล่นที่ใช้สกินของคุณอาจไม่ชอบถ้าคุณทำตัวเลขเริ่มต้นไว้แค่ครึ่งเดียว แล้วปล่อยอีกครึ่งให้กลับไปใช้ค่าเริ่มต้น กฎนี้บังคับใช้จริงจังเฉพาะกับสกินของบีตแมป แต่ถ้าทำสกินส่วนตัวก็ควรเอาไปพิจารณาด้วย

### สกินที่แก้จากของเดิมนับเป็น mixed skin ไหม?

สกินที่ถูกแก้ไขถือเป็น mixed skin ดังนั้นจึงโพสต์ใน subforum ใด ๆ ไม่ได้

### ทำสกิน remake จากสกินที่คนอื่นเคยทำได้ไหม?

ได้ คุณสามารถทำสกินธีมคล้ายกับสกินที่คนอื่นเคยทำมาก่อน แต่คุณต้องสร้าง asset เอง และแน่นอนว่าห้ามคัดลอก asset จากสกินที่คุณกำลังพยายาม remake

### ทำไมไม่มีฐานข้อมูลสกินเหมือนบีตแมป?

มีโปรเจกต์จาก third-party บางตัวที่พยายามทำสิ่งนี้อยู่แล้ว อย่างไรก็ตาม ฐานข้อมูลสกินถูกวางแผนไว้แล้ว ดูรายละเอียดเพิ่มเติมได้ที่ [ppy/osu-web#61 - Skin System](https://github.com/ppy/osu-web/issues/61)

## การแชร์

### แชร์สกินอย่างไร?

ก่อนจะแชร์สกิน ให้แน่ใจว่าคุณลบไฟล์อื่น ๆ ที่ไม่ได้ใช้ออกแล้ว เช่น รูปและเสียงเริ่มต้น โฟลเดอร์เสริม หรือไฟล์ที่ไม่ใช่รูปภาพ (ยกเว้นไฟล์ `skin.ini`) เมื่อเสร็จแล้ว คุณต้อง export สกินเป็นไฟล์ `.osk` จากนั้นอัปโหลดไฟล์นี้ แล้วโพสต์เกี่ยวกับสกินใน subforum skinning

1. เปิด osu!
2. เปิดแถบตัวเลือกด้านข้าง
3. โหลดสกินของคุณ
4. คลิก `Export as .osk`
5. จากนั้น osu! จะเปิดหน้าต่างที่เก็บสกินที่ export แล้วให้
6. นำไฟล์ `.osk` ที่มีชื่อสกินของคุณไปอัปโหลด
7. ตอนนี้คุณก็แชร์ได้แล้ว!

### v1.0 และ v2.0+ คืออะไร?

Skin v1.0 เป็นรูปแบบ skinning เก่าที่ใช้ตั้งแต่เกมเริ่มรองรับ skinning จนถึงเดือนมีนาคม 2013 หลังจากนั้น skin v2.0 ก็ถูกปล่อยออกมาพร้อมกับชุด skinning element แบบใหม่และการเปลี่ยนแปลงบางอย่าง หากต้องการดู changelog ของการอัปเดต skinning ให้ดูที่[ส่วน Versions ในบทความ skin.ini](/wiki/Skinning/skin.ini#versions)

การระบุว่าจะใช้เวอร์ชันไหนอาจดูไม่สำคัญ แต่ควรเข้าใจว่าฟีเจอร์เก่าบางอย่างอาจใช้ไม่ได้ใน skinning เวอร์ชันล่าสุด และในทางกลับกันก็เช่นกัน

## เสียง

### ลำดับความสำคัญของรูปแบบไฟล์เสียง

ลำดับรูปแบบไฟล์เสียงที่ osu! จะใช้มีดังนี้:

1. `.wav`
2. `.mp3`
3. `.ogg`

โดยพื้นฐานแล้ว osu! จะให้ความสำคัญกับไฟล์ `.wav` ก่อน เพราะไฟล์ชนิดนี้วนลูปเสียงสั้น ๆ ได้ดีกว่า `.mp3` หรือ `.ogg`

## รูปภาพ

### รูปภาพควรเป็นไฟล์รูปแบบไหน?

โดยส่วนใหญ่ รูปภาพต้องใช้รูปแบบ `.png` อย่างไรก็ตาม element `menu-background.jpg` ต้องใช้นามสกุล `.jpg`

นอกจากการใช้รูปแบบ `.png` แล้ว คุณควร:

- ตัดขอบหรือ crop รูปภาพทุกครั้งที่ทำได้ osu!client จะ render ทุก pixel ของรูป ซึ่งทำให้มีภาระงานมากขึ้น!
  - อย่างไรก็ตาม ในบางส่วนของรูป จำเป็นต้องมีความโปร่งใสบางจุดเพื่อจัดตำแหน่ง element บางชนิดให้ถูกต้อง
- บีบอัดรูปภาพทุกครั้งที่ทำได้ โดยปกติการบีบอัดจะลบข้อมูลที่ไม่จำเป็นเกี่ยวกับ pixel ว่าง ๆ และลดขนาดไฟล์ได้มาก

### Overlay คืออะไร?

Overlay คือ element ที่จะถูกวางทับบน element อีกชิ้นหนึ่ง (ชื่อ element เดียวกัน แต่ไม่มี suffix `overlay` หรือ `-overlay`) element กลุ่มนี้ค่อนข้างพิเศษ เพราะจะไม่ถูกเปลี่ยนสีหรือ tint อย่างไรก็ตาม มี element เพียงไม่กี่รายการเท่านั้นที่มี overlay (ดูได้ในรายการ skin element)

### ควรทำสกินสำหรับความละเอียดหน้าต่างเท่าไร?

เนื่องจาก osu! สามารถเล่นได้หลายความละเอียดหน้าต่าง skinning element บางอย่างอาจซ้อนทับกันหรือถูกวางห่างกว่าที่คาดไว้ ซึ่งหมายความว่าไม่ใช่ทุก element ที่จะยืดหรือ scale ตัวเองให้พอดีกับความละเอียดหน้าต่างเกม

ตอนทำ skinning ควรนึกถึงความละเอียดเหล่านี้ไว้:

- 1024x768 (4:3, ความละเอียดมาตรฐานของเกม และเกมอิงจากค่านี้)
- 2048x1536 (4:3, ความละเอียดมาตรฐานของเกมใน HD scaling)
- 1366x768 (16:9, ความละเอียดจอกว้างมาตรฐานของเกม)
- 2732x1536 (16:9, ความละเอียดจอกว้างมาตรฐานของเกมใน HD scaling)
- 1920x1080 (16:9, ความละเอียด HD มาตรฐาน)

เกมจะปรับรูปภาพให้เข้ากับความละเอียดที่แปลงมาจากค่าด้านบนเอง ส่วนใหญ่จะถูก rescale ให้พอดีกับเพลย์ฟีลด์ หรือถูกจัดตำแหน่งใหม่ใน aspect ratio ที่ต่างกัน

### รูปภาพ HD

รูปภาพ HD จะถูกใช้แทนรูปภาพปกติในความละเอียดสูง ถ้ามีไฟล์อยู่ เมื่อเทียบกับรูปปกติ รูป HD จะคมและสะอาดกว่ามากบนความละเอียดขนาดใหญ่เหล่านี้ เกมจะ scale รูปให้เข้ากับความละเอียดที่ใช้อยู่โดยอัตโนมัติ **ความละเอียดที่รองรับรูป HD ต้องมีความสูงอย่างน้อย 800 pixel** รูป HD จะมี suffix `@2x` ในชื่อไฟล์ เช่น `cursor@2x.png`

รูป HD มีขนาดมิติเพิ่มเป็นสองเท่า ตัวอย่างเช่น sprite ปกติ `hitcircle.png` มีขนาด 128x128 pixel ดังนั้น sprite HD ของมันคือ `hitcircle@2x.png` จะมีขนาด 256x256 pixel

sprite ทุกตัวมี counterpart แบบ HD แม้แต่ทุกเฟรมใน animation ก็มี counterpart แบบ HD ได้ ผลคือขนาดไฟล์ของโฟลเดอร์หรือ archive จะเพิ่มขึ้น เพราะมีรูปมากกว่าปกติ

รูป HD ทั้งหมดอาจมีขนาดไฟล์ใหญ่ขึ้นด้วย เพราะ canvas size ที่ใช้ใหญ่กว่าสprite ปกติถึงสี่เท่า โดยหลักแล้ว osu! ใช้โหมดความละเอียดอยู่สองแบบ แต่ละแบบจะเลือกใช้ชุดรูปที่ต่างกัน โหมดแรกคือ *LowResolution* ส่วนโหมดที่สองคือ *HighResolution*

- โหมด LowResolution ใช้รูปปกติและมองข้ามรูป HD (หรือเรียกว่าสกินความละเอียด SD)
- โหมด HighResolution ให้ความสำคัญกับรูป HD และใช้รูปปกติเป็น fallback ถ้าไม่มี sprite HD (หรือเรียกว่าสกินความละเอียด HD)

### ลำดับคะแนน hit score บนหน้าจอ ranking {#ranking-screen-hit-score-hierarchy}

หน้าจอ ranking จะให้ความสำคัญกับ skinning element แบบ static มากกว่าแบบ animation กล่าวคือ ถ้ามีทั้ง animation และเวอร์ชัน static ของ element เดียวกัน ลำดับความสำคัญจะเป็นดังนี้:

1. รูป static
2. เฟรมที่ศูนย์ของ animation
3. รูปเริ่มต้น

อย่างไรก็ตาม หน้าจอ ranking ของ osu!mania เป็นข้อยกเว้น กล่าวคือ ถ้ามีทั้ง animation และเวอร์ชัน static ของ element เดียวกัน ลำดับความสำคัญจะเป็นดังนี้:

1. เฟรมที่ศูนย์ของ animation ใน root directory
2. รูป static ใน root directory
3. รูปเริ่มต้น

*หมายเหตุ: ใน osu!mania, osu! จะไม่สน path ที่กำหนดไว้ในไฟล์ `skin.ini` และจะใช้เฉพาะรูปจาก root directory เท่านั้น (โฟลเดอร์ที่โหลด skin element เข้ามา)*

## Animation

### ทำให้ element เคลื่อนไหวอย่างไร?

อย่างแรก ให้ตรวจรายการ element ก่อนว่า element ที่คุณอยากทำให้เคลื่อนไหวนั้นทำ animation ได้จริงหรือไม่ ถ้าทำได้ ก็เริ่มทำ animation ได้เลย วิธีคือคุณต้องแยกแต่ละเฟรมของ animation ออกเป็นรูปคนละไฟล์ เมื่อทำเสร็จแล้ว ให้ตั้งชื่อแต่ละเฟรมด้วยชื่อ element ตามด้วยขีดกลาง (`-`) แล้วต่อด้วยเลขลำดับ index (เริ่มจาก 0) ทำซ้ำแบบนี้กับเฟรมที่เหลือ

ตัวอย่างเช่น ถ้าคุณต้องการทำ animation ให้ `scorebar-colour.png` ให้ตั้งชื่อเฟรมแรกเป็น `scorebar-colour-0.png` เฟรมที่สองเป็น `scorebar-colour-1.png` เฟรมที่สามเป็น `scorebar-colour-2.png` แล้วทำต่อไปเรื่อย ๆ

*หมายเหตุ: ไม่ใช่ทุก element ที่มีขีดกลาง (`-`) ระหว่างชื่อกับเลขเฟรม ตรวจรายการ element เพื่อดูชื่อ animation ให้ถูกต้อง*

frame rate ของ animation ในแต่ละ element ไม่เท่ากัน บางอันใช้ค่าที่กำหนดในไฟล์ skin.ini บางอันใช้ค่าที่ client กำหนดไว้ และบางอันอิงตาม BPM (beats per minute) ของบีตแมป เพราะแบบนี้ การทำ animation ให้ skinning element จึงต้องลองผิดลองถูก (ดูวิธีแก้ในหัวข้อถัดไป)

### Animation ของฉันเล่นเร็ว/ช้าเกินไป จะแก้อย่างไร?

ถ้า animation ของคุณเร็วหรือช้าเกินไป มีวิธีแก้สองแบบ:

1. ถ้ามันเล่นเร็วเกินไป ลองทำให้ยาวขึ้นด้วยการเพิ่มเฟรมซ้ำ (หรือถ้ามันช้าเกินไป ให้ลดเฟรมลงครึ่งหนึ่ง)
   - เช่น เฟรม 0 และ 1 เป็นรูปเดียวกัน (รูปนั้นจึงอยู่บนจอ 2 เฟรม), เฟรม 2 และ 3 เป็นรูปที่สอง เป็นต้น
   - วิธีนี้จะทำให้ animation ช้าลงเล็กน้อย (หรือเร็วขึ้นเล็กน้อย) แต่ rate ของ animation ยังเท่าเดิม หมายความว่าแม้คุณจะทำให้อันหนึ่งช้าลง/เร็วขึ้นแล้ว animation ของ element อื่นก็ยังอาจเร็วหรือช้าเกินไปอยู่ดี
2. ใช้คำสั่ง `AnimationFramerate` ใน `skin.ini`
   - คำสั่งนี้มีผลกับ animation ทั้งหมด ยกเว้นบางอย่าง เช่น `pippidon` หรือ `hitcircleoverlay`

## Skinning

### ทำ cursortrail แบบยาวได้อย่างไร?

คุณต้องมี element `cursormiddle.png` อยู่ในสกิน ถ้าไม่ต้องการให้เห็น ให้ใช้รูปโปร่งใสแทน

### ฉันลบ element ไปแล้ว แต่ยังเห็นอยู่! จะลบอย่างไร?

ถ้าต้องการลบ element คุณต้องใช้รูปโปร่งใสแทน element ที่ต้องการลบ จากนั้น[รีโหลดสกิน](#how-do-i-reload-my-skin-in-osu)

#### ลบแสงหลังจากกด object ได้อย่างไร?

มีสองวิธี:

- ปิด `Hit Lighting` ใน[ตัวเลือก](/wiki/Client/Options)
- ใช้รูปโปร่งใสชื่อ `lighting.png` ในสกินของคุณ

##### หมายถึง particle หลังคลิกต่างหาก!

สำหรับ particle ให้ลบไฟล์ `particle50.png`, `particle100.png` และ `particle300.png` ออกได้เลย ไม่จำเป็นต้องใช้รูปโปร่งใสสำหรับสามไฟล์นี้

#### ลบวงกลมปลายสไลเดอร์ได้อย่างไร?

เพิ่มไฟล์โปร่งใสชื่อ `sliderendcircle.png` ลงในสกิน

#### ซ่อน hit score 300 ได้อย่างไร?

เพิ่มไฟล์โปร่งใสสำหรับชื่อไฟล์เหล่านี้:

- `hit300-0.png`
- `hit300g-0.png`
- `hit300k-0.png`

### ใส่พื้นหลังใน main menu ได้อย่างไร?

คุณต้องมี [osu!supporter](/wiki/osu!supporter) ถึงจะเห็นรูปพื้นหลังใน main menu ได้ หากต้องการเปลี่ยนอย่างรวดเร็ว คุณสามารถลากและวางรูป `.png` หรือ `.jpg` ลงใน osu! (ตอนอยู่หน้า main menu) วิธีนี้จะเขียนทับรูปปัจจุบันในโฟลเดอร์ของสกินที่เลือกอยู่

### ปิด hitsound หนึ่งรายการหรือมากกว่านั้นอย่างไร?

hitsound มีสามชนิดที่ใช้ prefix เหล่านี้: `normal`, `soft` และ `drum` โดยทุกชนิดมี suffix: `-hitnormal`, `-hitwhistle`, `-hitfinish` และ `-hitclap`

ถ้าต้องการปิด ให้หาเสียงที่คุณไม่ชอบแล้วแทนที่ด้วยเสียงอื่น
