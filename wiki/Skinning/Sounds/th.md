# เสียงที่ทำสกินได้

รูปแบบที่ใช้ได้สำหรับเสียงคือ `.wav`, `.mp3` และ/หรือ `.ogg`

## อินเทอร์เฟซ

### Main menu

| Filename | Usage |
| :-- | :-- |
| `heartbeat.wav` | เมื่อเคอร์เซอร์ hover เหนือ osu! cookie ใน main menu |
| `seeya.wav` | ตอนปิด osu! client |
| `welcome.wav` | ตอนเริ่ม osu! client |

Notes:

- สำหรับ `seeya.wav` และ `welcome.wav`:
  - ต้องมี [osu!supporter](/wiki/osu!supporter) จึงจะแก้ไขได้
  - สามารถปิดได้โดยเอาเครื่องหมายถูกออกจาก `osu! music theme` ใน[ตัวเลือก](/wiki/Client/Options)

## Gameplay

เสียง gameplay ต่อไปนี้สามารถทำสกินผ่านบีตแมปได้ทั้งหมด

### Countdown

- `count.wav`:
  - ใช้แทนเสียง `count{n}s` (ถ้าต้องการใช้เสียงเดียว)
  - ยังเป็น metronome ของม็อด [Target Practice](/wiki/Gameplay/Game_modifier/Target_Practice) ด้วย
- `count1s.wav`
- `count2s.wav`
- `count3s.wav`
- `gos.wav`
- `readys.wav`

### Playfield

| Filename | Usage |
| :-- | :-- |
| `comboburst.wav` | ทำคอมโบถึง milestone |
| `combobreak.wav` | คอมโบแตกที่คอมโบ 20 หรือมากกว่า |
| `failsound.wav` | เล่นไม่ผ่านแมปในช่วงใดก็ตาม |
| `sectionpass.wav` | ผ่านครึ่งหนึ่งของ break โดยมี HP เหลือสูง |
| `sectionfail.wav` | ผ่านครึ่งหนึ่งของ break โดยมี HP เหลือต่ำ |
| `fountain-shoot.wav` | ถึงจุดเริ่มต้นของช่วง kiai |

Notes:

- `comboburst.wav`:
  - ถ้าต้องการใช้หลายเสียง ให้ใช้ `comboburst-{n}.wav`
  - ใน [skin.ini](/wiki/Skinning/skin.ini) หากตั้งค่า `CustomComboBurstSounds` เป็น `1` จะต้องใช้ `comboburst-{n}.wav`
- `failsound.wav`:
  - ไม่ควรยาวเกิน animation ตอน fail (ประมาณ 5 วินาที)
  - การใช้ track เสียงที่ยาวจะกระทบ performance ของ game client เพราะมันถูกโหลดทุกครั้งที่คุณเล่นบีตแมป แม้คุณจะไม่เคย fail ก็ตาม ไฟล์นี้ไม่ควรใหญ่เกินไม่กี่กิโลไบต์
- `fountain-shoot.wav`:
  - เสียงนี้ใช้เฉพาะในเกมเวอร์ชัน [Lazer](/wiki/Client/Release_stream/Lazer) และไม่มีผลใน osu!(stable)
  - เสียงจะเล่นพร้อม animation น้ำพุที่จุดเริ่มต้นของแต่ละช่วง kiai ในแมป
  - เสียงนี้จะเล่นใน main menu ด้วยเมื่อ client อยู่ในโฟกัส

## Rank display (เฉพาะ Lazer)

ในเกมเวอร์ชัน [Lazer](/wiki/Client/Release_stream/Lazer) มีการเพิ่ม component rank display ใหม่สำหรับทำสกินใน osu!editor โดยสามารถใช้ audio element ต่อไปนี้ทำสกินให้ component เหล่านี้ได้ และไม่มีผลใน osu!(stable)

เสียง rank display จะเล่นก็ต่อเมื่อมีการเพิ่ม component rank display อย่างน้อยหนึ่งตัวในสกินปัจจุบันของคุณ

| Filename | Usage |
| :-- | :-- |
| `rank-up.wav` | ได้ rank สูงขึ้น |
| `rank-down.wav` | ตกลงไปเป็น rank ต่ำกว่าเดิม |

Notes:

- `rank-down.wav` จะเล่นทันทีหลังจากคุณเสีย rank ตัวอักษรสูงสุด แต่การเปลี่ยน rank ถัด ๆ ไปอาจถูกหน่วงได้สูงสุด 1.5 วินาที

### Game screens

เสียงเหล่านี้ไม่ควรใหญ่เกินไม่กี่กิโลไบต์ เพราะการใช้ track เสียงที่ยาวจะกระทบ performance ของ game client

`applause.wav`:

- เสียงนี้จะเล่นบนหน้าจอ ranking หลังจากเคลียร์แมป
- ควรจัดรูปแบบเป็น `.mp3` หรือ `.ogg` ในบีตแมปที่ตั้งใจส่งจัดอันดับ

ในเกมเวอร์ชัน [Lazer](/wiki/Client/Release_stream/Lazer) คุณสามารถตั้งได้ว่า audio ไหนจะเล่นตาม letter grade แต่สิ่งเหล่านี้ไม่มีผลใน osu!(stable) และ `applause.wav` จะถูกเล่นเสมอในเวอร์ชันนั้น

`applause-{g}.wav`

Notes:

- `{g}` สามารถแทนด้วย letter grade `S`, `A`, `B`, `C` หรือ `D`
- คุณไม่สามารถเล่นเสียงใด ๆ สำหรับ letter grade `F` ได้ เพราะเมื่อ fail จะไม่มีทางไปถึง result screen

`pause-loop.wav`:

- เสียงนี้จะเล่นเมื่อเกมถูก pause
- เสียงนี้จะถูก loop
- fade out เมื่อ client หลุดจากโฟกัส

#### Spinner set

`spinnerspin.wav`:

- เสียงนี้จะถูก loop
- เสียงนี้อาจ modulate จาก ~500 hz ถึง 80000 hz
  - หากต้องการปิด ให้ตั้งค่า `SpinnerFrequencyModulate` เป็น `0`

`spinnerbonus.wav`:

- เล่นทุกครั้งที่คุณได้ bonus 1000 คะแนนจากการหมุน

`spinnerbonus-max.wav`:

- ใน client [Lazer](/wiki/Client/Release_stream/Lazer) เสียงนี้จะเล่นแทน `spinnerbonus.wav` ถ้า spinner ได้คะแนนสูงสุดไปแล้ว และไม่มีผลใน osu!stable

#### ม็อด Nightcore

*หมายเหตุ: ไฟล์จะถูกดึงจากสกินของคุณ ไม่ใช่สกินของบีตแมป*

เสียงเหล่านี้จะเล่นทับเพลงเมื่อเปิดใช้งาน[ม็อด Nightcore](/wiki/Gameplay/Game_modifier/Nightcore) ในจังหวะ 4/4:

- `nightcore-kick.wav` จะเล่นบน[beat](/wiki/Music_theory/Beat) ที่ 1 และ 3
- `nightcore-clap.wav` จะเล่นบน beat ที่ 2 และ 4
- `nightcore-hat.wav` จะเล่นบน [quaver](https://en.wikipedia.org/wiki/Eighth_note) ลำดับคี่ทุกตัว แต่เฉพาะเมื่อ [slider tick rate](/wiki/Beatmapping/Slider_tick_rate) ของบีตแมปเป็นผลคูณของ 2
- `nightcore-finish.wav` จะเล่นบน beat แรกของทุก 4 [measure](/wiki/Music_theory/Measure) เว้นแต่ barline นั้นถูกละไว้ด้วย [timing point](/wiki/Client/Beatmap_editor/Timing#uninherited-timing-point)
