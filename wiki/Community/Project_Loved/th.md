# Project Loved

**Project Loved** คือแพลตฟอร์มการลงคะแนนเพื่อผลักดัน [Beatmap](/wiki/Beatmap) เข้าสู่หมวดหมู่ [Loved](/wiki/Beatmap/Category#loved) โดยดำเนินงานหลักโดยทีม [Project Loved Team](/wiki/People/Project_Loved_Team)

## การย้ายบีทแมพเข้าสู่หมวด Loved

บีทแมพจะถูกย้ายเข้าสู่หมวด Loved ผ่านกระบวนการเสนอชื่อและการลงคะแนนเสียง

สมาชิกของทีม Project Loved Team ที่เรียกว่า **กัปตัน (Captains)** จะทำการเสนอชื่อบีทแมพสำหรับแต่ละโหมดเกมเป็นระยะ โดยแมพเหล่านี้เกือบทั้งหมดมาจากการส่งชื่อโดยคนในชุมชน osu!

หากคุณต้องการส่งชื่อบีทแมพเพื่อให้พิจารณาเข้าหมวด Loved โปรดไปที่ [หน้าส่งรายชื่อบีทแมพ](https://loved.sh/submit) บนเว็บไซต์ Project Loved และกรอกแบบฟอร์ม รายชื่อที่ถูกส่งเข้ามาสามารถดูได้ในหมวดหมู่ของแต่ละโหมดเกมตามลิงก์ด้านล่าง:

- [รายการส่งชื่อโหมด osu!](https://loved.sh/submissions/osu)
- [รายการส่งชื่อโหมด osu!taiko](https://loved.sh/submissions/taiko)
- [รายการส่งชื่อโหมด osu!catch](https://loved.sh/submissions/fruits)
- [รายการส่งชื่อโหมด osu!mania](https://loved.sh/submissions/mania)

บีทแมพที่ได้รับการเสนอชื่อจะถูกนำไปโพสต์ใน [ฟอรัม Project Loved](https://osu.ppy.sh/community/forums/120) เพื่อให้สมาชิกในชุมชนได้ร่วมลงคะแนนว่าต้องการให้แมพนั้นอยู่ในหมวด Loved หรือไม่ แมพที่ได้รับคะแนนเสียง "เห็นด้วย (Yes)" ถึง 80% เมื่อสิ้นสุดระยะเวลาการลงคะแนน จะถูกย้ายเข้าสู่หมวดหมู่ Loved ทันที

เจ้าของผลงานแมพเหล่านั้น **ไม่สามารถ** ร้องขอให้นำแมพออกจากหมวด Loved ได้ นอกจากนี้ ระดับความยากที่อยู่ในหมวด Loved จะไม่สามารถนำกลับไปใช้ในบีทแมพหมวด Ranked ได้โดยตรง

อย่างไรก็ตาม แมพอาจถูกถอดออกจากหมวดหมู่ชั่วคราวได้หากมีการค้นพบปัญหาที่ส่งผลเสียต่อการเล่นอย่างรุนแรงทันทีหลังจากที่ถูกเพิ่มเข้าไป คุณสามารถรายงานปัญหาเหล่านี้ได้โดยการติดต่อสมาชิก [Project Loved Team](https://osu.ppy.sh/groups/31) หรือโพสต์ในแชนแนล `#osu-loved` บน [เซิร์ฟเวอร์ Discord ของ osu!](https://discord.gg/ppy)

## คุณสมบัติเบื้องต้น (Eligibility)

มีเกณฑ์ขั้นต่ำบางประการที่บีทแมพต้องมีเพื่อให้ได้รับการเสนอชื่อเข้าสู่หมวด Loved โดยแต่ละโหมดเกมอาจมีเกณฑ์เฉพาะเจาะจงเพิ่มเติม แต่ทุกโหมดต้องปฏิบัติตามกฎพื้นฐานดังนี้:

- ปฏิบัติตาม [กฎการส่งบีทแมพ (Beatmap submission rules)](/wiki/Rules#beatmap-submission-rules)
- ไม่อยู่ในหมวดหมู่ [Ranked](/wiki/Beatmap/Category#ranked), [Approved](/wiki/Beatmap/Category#approved) หรือ [Qualified](/wiki/Beatmap/Category#qualified)
- ผู้สร้างผลงานต้องไม่คัดค้านการนำแมพเข้าหมวด Loved
  - หากมีผู้มีส่วนร่วมคนอื่น (Guest contributors) คัดค้าน ส่วนของพวกเขาจะถูกนำออกจากการเป็นหมวด Loved ด้วยวิธีที่เหมาะสม ซึ่งในบางกรณี เช่น [แมพคอลแลบ (Collabs)](/wiki/Beatmap/Beatmap_collaborations) อาจทำให้ทั้งชุดแมพไม่ผ่านเกณฑ์
- ผู้สร้างผลงานต้องไม่ถูก [ระงับบัญชี (Restricted)](/wiki/Help_centre/Account_restrictions)
- ต้องมีอย่างน้อยหนึ่งระดับความยากที่สามารถเล่นผ่านได้
- ทุกระดับความยากที่ได้รับการเสนอชื่อต้องมี [เวลาเล่นจริง (Drain time)](/wiki/Beatmap/Drain_time) อย่างน้อย 20 วินาที

แม้ว่าจะมีบีทแมพ Unranked จำนวนมากที่ผ่านเกณฑ์เหล่านี้ แต่มีเพียงส่วนน้อยเท่านั้นที่ได้รับเลือกเข้าสู่หมวด Loved เนื่องจากการคัดเลือกอย่างเข้มงวดของกัปตันและความต้องการคะแนนเห็นชอบในระดับสูงจากชุมชน

### การอัปเดตก่อนเข้าสู่หมวด Loved

ในบางกรณี บีทแมพอาจได้รับการเสนอชื่อแม้จะผิดเกณฑ์บางอย่าง ตราบใดที่ Mapper สามารถอัปเดตแมพให้ถูกต้องตามเกณฑ์ก่อนที่จะเข้าสู่หมวด Loved จริง เช่น แมพที่มีภาพพื้นหลังไม่เหมาะสมอาจได้รับการเสนอชื่อ หากมีการนำภาพนั้นออกหรือเปลี่ยนใหม่ในภายหลัง สมาชิกของ Project Loved มักจะโพสต์แจ้งในหน้า [พูดคุยของบีทแมพ (Discussion)](/wiki/Beatmap_discussion) หากจำเป็นต้องมีการแก้ไข

นอกจากนี้ สมาชิก Project Loved มักจะให้คำแนะนำด้านเทคนิค เช่น การปรับปรุงคุณภาพเสียง, ความแม่นยำของ [จังหวะ (Timing)](/wiki/Beatmapping/Timing) หรือการปรับ [Metadata ให้เป็นไปตามมาตรฐาน](/wiki/Ranking_criteria/Metadata) แม้ว่าจะไม่มีการบังคับ แต่ผู้สร้างแมพควรปฏิบัติตามคำแนะนำเหล่านี้เพื่อคุณภาพที่ดีของผลงาน

### เกณฑ์เฉพาะสำหรับโหมด osu!

บีทแมพในโหมด osu! จะมีการพิจารณาตามหมวดหมู่และกฎเกณฑ์ด้านล่าง ซึ่งคล้ายกับ [เกณฑ์การพิจารณา Ranked (Ranking criteria)](/wiki/Ranking_criteria) แต่มีการปรับให้ยืดหยุ่นมากขึ้นตามสไตล์ของ Project Loved

#### การจัดหมวดหมู่ (Categorisation)

ในทุกๆ เดือน บีทแมพจะถูกเลือกตาม 9 หมวดหมู่ เพื่อให้มั่นใจว่ารายชื่อแมพที่กัปตันเลือกมีความหลากหลาย ระบบนี้เริ่มใช้ตั้งแต่รอบเดือนกุมภาพันธ์ 2023 เป็นต้นมา

| หมวดหมู่ | คำอธิบาย |
| :-: | :-- |
| Popular Playcounts | บีทแมพยอดนิยมที่มีจำนวนครั้งการเล่นอย่างน้อย 100,000 ครั้ง |
| Outstanding Oldies | บีทแมพชั้นยอดที่ถูกส่งเข้ามาในช่วงครึ่งแรกของประวัติศาสตร์ osu! |
| Small Spectacles | บีทแมพที่โดดเด่นจากการพิจารณาของกัปตัน แต่มีจำนวนครั้งการเล่นน้อยกว่า 5,000 ครั้ง |
| Daredevil Difficulties | บีทแมพที่ขึ้นชื่อเรื่องความยากระดับสุดขีด เพื่อส่งเสริมการแข่งขันในหมู่ผู้เล่นระดับท็อป |
| Ranked Rejects | ชุดแมพที่มีระดับความยากครบถ้วนตามเกณฑ์การจัดอันดับ แต่ไม่เคยได้รับสถานะ Ranked |
| Gameplay Gimmicks | บีทแมพที่มีลูกเล่นการเล่นที่แปลกใหม่ เช่น ผ่าน Storyboard หรือการทำแมพสไตล์ Aspire |
| Meme Monstrosities | บีทแมพที่ได้รับความนิยมจากเนื้อหาที่ตลกขบขัน หรือเกี่ยวข้องกับวัฒนธรรมย่อยในชุมชน |
| Second Shots | บีทแมพที่เคยแพ้การโหวตในอดีต และได้รับโอกาสครั้งที่สอง |
| Chill Champions | บีทแมพสำหรับเล่นผ่อนคลาย โดยไม่มีระดับความยากไหนเกิน 5 ดาว |

นอกจากนี้ อาจมีการเลือกบีทแมพแบบ *Wildcard* เพิ่มเติมได้สูงสุด 3 แมพจากหมวดหมู่ใดก็ได้ ทำให้ในแต่ละเดือนมีแมพรวมทั้งหมด 12 แมพ

หากมีคำถามเกี่ยวกับหมวดหมู่เหล่านี้ สามารถติดต่อ [กัปตัน osu! Loved](https://osu.ppy.sh/groups/31?mode=osu) ได้โดยตรง

#### กฎระเบียบ (Rules)

**กฎเหล่านี้คือข้อบังคับที่ห้ามละเมิดโดยเด็ดขาด**

- **บีทแมพต้องถูกส่งเข้ามาอย่างน้อย 1 ปีก่อนได้รับการเสนอชื่อ ยกเว้นแมพที่ผิดเกณฑ์ปกติเพราะคอนเซปต์เฉพาะทาง** ไม่แนะนำให้จงใจทำแมพมาเพื่อลง Loved โดยเฉพาะหากแมพนั้นสามารถทำให้เป็น Ranked ได้ดีกว่า ข้อยกเว้นรวมถึงแมพที่ใช้ Sliderator, แมพ Aspire, หรือแมพที่มีลูกเล่น Storyboard
- **บีทแมพต้องมีจำนวนรายการโปรด (Favourites) อย่างน้อย 30 รายการ** เพื่อแสดงให้เห็นว่าชุมชนมีการตอบรับในเชิงบวกต่อแมพนั้นจริง
- **ผู้ที่ส่งชื่อแมพต้องเป็นผู้ที่มีส่วนร่วมในการทำแมพนั้นด้วยตนเอง** เพื่อป้องกันการนำ "แมพรวมเพลง (Compilations)" ที่ไม่ใช่ผลงานต้นฉบับมานำเสนอ
- **จังหวะ (Timing) ของแมพต้องไม่ส่งผลเสียต่อการเล่น** แม้จะไม่ต้องแม่นยำระดับมิลลิวินาทีเหมือนแมพ Ranked แต่ต้องอยู่ในระดับที่เหมาะสม
- **แมพที่มีลูกเล่นอิงจาก Storyboard ต้องไม่เปิดโอกาสให้ผู้เล่นโกงได้ง่ายๆ ผ่านการแก้ไขไฟล์** กัปตันจะเป็นผู้พิจารณาเป็นกรณีไปเพื่อให้เกิดความยุติธรรมในการแข่งขัน

#### แนวทางปฏิบัติ (Guidelines)

**แนวทางปฏิบัติอาจถูกละเว้นได้ในกรณีพิเศษที่มีเหตุผลสมควร**

- **กฎและแนวทางปฏิบัติของแมพ Ranked ที่ไม่ได้ระบุไว้ในนี้ ให้ถือเป็นแนวทางปฏิบัติทั้งหมด** โดยจะมีการพิจารณาเป็นกรณีไปและจะติดต่อ Mapper หากจำเป็นต้องมีการแก้ไข
- **Metadata ของบีทแมพควรมีความถูกต้องมากที่สุด** ก่อนการโหวตจะมีการตรวจสอบความถูกต้องของข้อมูลเพลงโดยผู้เชี่ยวชาญเพื่อให้ง่ายต่อการค้นหา
- **จุดจังหวะแบบ Uninherited (เส้นแดง) ควรจะเหมือนกันในทุกระดับความยาก** อาจยกเว้นได้สำหรับลูกเล่นบางอย่างที่จงใจเปลี่ยน BPM หรือความเร็ว Slider
- **ทุกระดับความยากในชุดแมพควรจะทำเสร็จสมบูรณ์** หมายถึงมีจุดเริ่มและจุดจบของเพลงที่สอดคล้องกัน ไม่ควรมีระดับความยากไหนที่สั้นกว่าอันอื่นอย่างผิดปกติ
