# 设置 osu!tourney

*另见：[osu! 比赛客户端](/wiki/osu!_tournament_client)*\
*注意：目前需要有效的 [osu! 支持者标签](/wiki/osu!supporter)才能使用 osu!tourney 客户端。*

建议**安装一个全新的 osu! 客户端**。这是因为歌曲数据库极有可能被破坏。你可以同时安装多个 osu! 客户端，只需要将它们存放在不同的文件夹即可。

如果你要这么做，请将目录下的 `osu!.exe` 复制到一个新的空文件夹，然后运行程序，它应该会自动开始更新和安装。在此之后，只需要对这个客户端做修改就行了。

打开安装完毕的 osu! 并登录。不要忘记选择 `记住用户名` 和 `记住密码`。

然后关闭 osu!，在这个新的文件夹里创建一个名为 `tournament.cfg` 的文件。重新启动你的 osu! 客户端，osu!tourney 应该会启动。

在启动客户端之前，你需要将图池放入 `Songs` 文件夹中。客户端会自动下载缺失的*已提交*谱面。

关闭客户端时，会自动生成 `tournament.cfg` 文件的内容，它包含以下行：

```
TeamSize = 4
acronym = OWC
```

不要在 osu! 客户端运行时修改 `tournament.cfg` 文件。请先关闭 osu!，然后编辑或添加所需配置：

- `TeamSize`：每个队伍的玩家数目。
- `ClientNameSize`：玩家名称字体大小。
- `privateserver`：不要改变这个选项，请保持原样。
- `acronym`：比赛名的缩写。这会影响比赛房间所需命名，详情请见[多人游戏房间管理](/wiki/osu!_tournament_client/osu!tourney/Multiplayer_usage)。
- `BufferTimeoutTime`：客户端等待玩家缓冲的时间，超时后将同步剩余玩家显示。
  - 不建议修改此项。
- `BufferTotalTime`：客户端缓冲的时间，有助于降低等待缓冲的频率。这会使客户端与玩家之间产生延迟。
  - 不建议修改此项。
- `BufferDangerTime`：客户端在暂停等待更多的回放帧之前，回放缓冲区的剩余时间。
  - 不建议修改此项。
- `RankingChatDelay`：回合结束后隐藏计分板并显示聊天的延迟（以毫秒为单位）。
- `Height`：客户端观战区域的高度。
  - 客户端会以 16:9 的比例自动计算宽度。
  - 控制面板不包含在“观战区域”内。一般情况下控制面板会比客户端高 200 像素，所以你其实需要 1440p 的屏幕来直播 1080p 的比赛。
  - 默认值：720，允许范围：568-2160
- `Aspect`：游玩区域显示区的宽高比。
  - 请注意，此配置仅用于 2v2 布局。
  - osu!taiko 适用的理想数值为 1.5。
  - 默认值：2，允许范围：1-2
