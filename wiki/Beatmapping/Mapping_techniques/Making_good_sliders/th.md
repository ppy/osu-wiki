# Making good sliders

บทความนี้เป็น how-to สำหรับการทำสไลเดอร์ดี ๆ
ในรูปแบบพื้นฐานที่สุด สิ่งที่คุณต้องมีคือ [bézier curve](https://en.wikipedia.org/wiki/B%C3%A9zier_curve)

จาก [Ephemeral](https://osu.ppy.sh/users/102335),

> จัด endpoint ของสไลเดอร์ให้ตรงกับค่าที่เล็กที่สุดเท่าที่ทำได้เสมอ หรือก็คือถ้าลากถอยหลังไปไกลกว่านั้นอีก สไลเดอร์จะสั้นลง point จะจัดตัวเองให้อยู่กลาง slider end และทำให้การทำ curve ที่พอใช้ได้ง่ายขึ้นมาก

และการทำแบบนี้ยังทำให้ reverse selection บนสไลเดอร์ไม่ทำให้มันพังด้วย
ซึ่งเป็นเรื่องดี

## Arcs

ใช้เทคนิคเดียวกันโดยพื้นฐานกับสไลเดอร์ใด ๆ ที่สมมาตรในหนึ่งแกน

![ตัวอย่าง arcs](img/MGS_arcs.png)

นี่คือรูปร่างที่ทำง่ายที่สุด
โดยพื้นฐานแล้ว คุณเริ่มจากวาง points ทั้งหมดให้สมมาตรบนกริด
เลือกมันแล้วกด `Ctrl` + `H` เมื่อคิดว่ามันแม่นแล้ว เพื่อให้แน่ใจว่า points อยู่บนกริดที่สมมาตร
จากนั้นขยับ points ไปมาจนสไลเดอร์มีความยาวถูกต้อง และหยุดก่อนถึง point สุดท้ายเล็กน้อย
สิ่งสำคัญคือทุกครั้งที่คุณขยับ point หนึ่ง ต้องขยับ point ที่สัมพันธ์กันในแบบเดียวกันเป๊ะ ๆ เพื่อรักษาให้ points ทั้งหมดสมมาตรกัน
อย่าลืมกด `Ctrl` + `H` บ่อย ๆ เพื่อเช็กว่าไม่ได้ทำพลาด

เพื่อให้ slider-end กับ endpoint ตรงกัน ให้กด shift ค้างเพื่อปิด grid snap แล้วขยับ point ด้านบนลงมาจน slider-end อยู่ตรงบน point สุดท้ายพอดี
เมื่อคิดว่าสมบูรณ์แล้ว ให้เลือกและกด `Ctrl` + `H` สองสามครั้ง แล้วดูว่า endpoint ขยับหรือไม่
ถ้าไม่ขยับ ยินดีด้วย คุณมีสไลเดอร์สวย ๆ แล้ว
เอาไปอวดเพื่อนได้เลย
ถ้ามันขยับ ให้ปรับ point ด้านบนไปมาจนกว่าจะไม่ขยับ

ถ้าต้องการ arc ที่เอียง โดยปกติวิธีที่ง่ายที่สุดคือทำให้ตั้งตรงแบบด้านบนก่อน แล้วไปที่ `Edit` จากนั้น `Rotate By...` เพื่อหมุนให้ได้มุมที่ต้องการ

## Waves

ใช้เทคนิคเดียวกันโดยพื้นฐานกับสไลเดอร์ใด ๆ ที่มี rotational symmetry

![ตัวอย่าง waves](img/MGS_waves.png)

Waves ทำคล้ายกับ arcs มาก เพียงแต่ points ของคุณควรสมมาตรแบบหมุนเมื่อเทียบกัน ไม่ใช่แค่ flip แนวนอน
ถ้ากด `Ctrl` + `H` + `J` มันจะหมุนสไลเดอร์ 180 องศา เพื่อให้คุณเช็กได้ว่า points ตรงกัน

เพื่อให้ slider-end กับ endpoint ตรงกัน ให้เลือก points สมมาตรที่สัมพันธ์กันสองจุด
ปิด grid snap แล้วขยับทั้งสองจุดเข้าด้านในทีละนิด จน slider-end และ point สุดท้ายอยู่ทับกันพอดี

## Beat Blankets

*หน้าหลัก: [Blanket Combos](/wiki/Beatmapping/Mapping_techniques/Formations#blanket-combos)*

![ตัวอย่าง beat blankets (พร้อมฮิตเซอร์เคิล)](img/MGS_blankets.png)

Blankets ทำง่ายที่สุดถ้าคุณกะสไลเดอร์ด้วยตาก่อนให้ถูกคร่าว ๆ จากนั้นเลื่อนไปยังจุดบนไทม์ไลน์ที่ approach circle ของ beat ตรงกับขอบที่ใกล้ที่สุดของ startpoint ของสไลเดอร์
การเข้า snap 1/8 อาจช่วยให้ทำได้พอดีขึ้น
จากนั้นปรับ points ไปเรื่อย ๆ จนสไลเดอร์ทั้งตัวตรงกับ approach circle ให้สมบูรณ์ที่สุดเท่าที่ทำได้

จำไว้ว่าถ้าต้องการให้สไลเดอร์ blanket **และ** สมมาตรเป๊ะ ให้จัดการ symmetry ก่อน แล้วค่อยปรับ points แบบสมมาตรเพื่อทำให้ blanket ดีขึ้น

> บางครั้งผมใช้สปินเนอร์เพื่อทำวงกลมสวย ๆ เป็น variation ของ beat blankets

## Circles

ขอบคุณ mm201

![ตัวอย่าง circles](img/MGS_circles.jpg)

จำนวน control points ที่ต้องใช้ขึ้นอยู่กับมุมที่ arc ของคุณครอบคลุม:

- 0 องศา: 2 points
- 0-20 องศา: 3 points
- 20-170 องศา: 4 points
- 170-200 องศา: 5 points
- 200-300 องศา: 6 points
- 300-350 องศา: 7 points

ทั้งหมดนี้ไม่ได้แม่นเป๊ะ เป็นแค่ไอเดียทั่วไป
คล้ายกับ waves ยิ่ง control point อยู่ไกลจาก start/end ตามลำดับมากเท่าไร มันก็ต้องอยู่ห่างจาก curve มากขึ้นเท่านั้น
ตรวจให้แน่ใจว่า control lines แรกและสุดท้ายชี้ไปในทิศทางที่คุณต้องการให้ loop เริ่ม
ส่วนที่เหลือให้ปรับด้วยสายตาจนส่วนใหญ่ดูกลม
การใช้ approach circle เป็นไกด์ช่วยได้

เหมือนกับรูปร่างส่วนใหญ่ ให้เหลือ control point ไว้หนึ่งจุด (สองจุดถ้าจำนวน points เป็นเลขคู่) เพื่อเอาออกจาก grid snap จะได้จัด endpoint ให้ตรงกัน

## Elbows

ใช้เทคนิคเดียวกันกับสไลเดอร์ใด ๆ ที่มีทั้งส่วนตรงและส่วนโค้ง

![ตัวอย่าง elbows](img/MGS_elbows.png)

เมื่อคุณต้องการสไลเดอร์ที่มี transition ระหว่างส่วนโค้งกับส่วนตรง ให้ใช้ red point ตรงจุดที่เปลี่ยน
สิ่งสำคัญที่สุดคือ **ต้องวาง red point และ points ทั้งสองด้านให้อยู่ในเส้นตรงเสมอ**
ต้องตรงจริง ๆ
ถ้าจำเป็นก็หาไม้บรรทัดมาใช้
เราไม่ต้องการมุมหักคม ๆ ตรงนี้
นอกเหนือจากนั้นก็ตรงไปตรงมาพอสมควร

![ตัวอย่าง hearts](img/MGS_hearts.png)

## Wiggles

มีสองสามวิธีในการทำสิ่งนี้ ขึ้นอยู่กับว่าคุณอยากได้ wiggle แบบไหน

### Type 1

จุดเริ่มและจุดจบชี้ไปทางเดียวกัน

![ตัวอย่าง wiggles (type 1)](img/MGS_wiggles1.png)

สิ่งสำคัญตรงนี้คือ arc แต่ละช่วงที่ประกอบเป็น wiggle ต้องประกอบด้วย points สี่จุด และยิ่งเข้าใกล้ตรงกลาง points สี่จุดนั้นยิ่งต้องสูงขึ้น
เมื่อมีรูปร่างพื้นฐานแล้ว ก็เป็นการลองปรับและกะด้วยสายตาจนมันดูเท่ากัน และปลาย tail ของสไลเดอร์ตรงกับ point สุดท้าย
เหมือนกับ arcs อย่าลืมกด `Ctrl` + `H` ตลอดเวลาเพื่อให้แน่ใจว่า points สมมาตร
นอกเหนือจากนั้น ส่วนใหญ่คือการฝึก
เมื่อทำ wiggles ที่ดูดีได้สักสองสามอันแล้ว คุณจะทำมันได้เร็วขึ้นมาก

### Type 2

จุดเริ่มและจุดจบชี้คนละทาง

![ตัวอย่าง wiggles (type 2)](img/MGS_wiggles2.png)

คล้ายกับ [type 1](#type-1) arc แต่ละช่วงประกอบด้วย points สี่จุด แต่จากที่แมปเปอร์มีประสบการณ์บอกไว้ มันจะช่วยถ้า points บิดวนเล็กน้อยเหมือนในภาพด้านบน
สิ่งเหล่านี้มี rotational symmetry เหมือน waves ดังนั้นใช้ `Ctrl` + `H` + `J` เพื่อเช็ก points

### Type 3

Super Tight Wiggles

![ตัวอย่าง wiggles (type 3)](img/MGS_wiggles3.png)

บางครั้ง wiggles แน่นเกินกว่าจะใช้วิธีด้านบนได้ จึงจำเป็นต้องใช้ red-points
ตรวจให้แน่ใจว่า points ของแต่ละ section ระหว่าง red-points แต่ละคู่เหมือนกัน ยกเว้นส่วนท้าย
คุณสามารถเช็กได้ด้วยการ duplicate สไลเดอร์แล้วขยับไปมา ดูว่ามันเรียงตรงกันตลอดแนวหรือไม่ หรือแค่นับกริดก็ได้
นอกจากนี้ ตรวจให้แน่ใจว่า redpoint แต่ละจุดและ points สองจุดด้านข้างสร้างเส้นตรงสมบูรณ์ เพื่อหลีกเลี่ยง bumps ที่ไม่ต้องการ
Sections ช่วงท้ายต้องกะด้วยตาจริง ๆ และควรเป็นส่วนที่คุณปรับเมื่ออยากให้ tail end ลงบน point สุดท้าย
นี่ต้องลองปรับเยอะมาก แต่หวังว่าสุดท้ายคุณจะทำได้

## Loops

สไลเดอร์ประเภทพูดง่ายแต่ทำยาก

![ตัวอย่าง loops](img/MGS_loops.png)

สิ่งที่ต้องจำตอนสร้าง loops คือ points จะสูงกว่า loop จริงค่อนข้างมาก:

- ยิ่ง point แต่ละจุดอยู่ไกลจากปลายมากเท่าไร มันก็ยิ่งต้องอยู่ห่างจากสไลเดอร์มากขึ้นเท่านั้น

ปัญหาใหญ่ที่สุดที่มักเห็นใน loops คือรู
สิ่งที่คุณต้องการคือรูปหยดน้ำที่เปิดและโค้งมนเหมือนสไลเดอร์ด้านบน
ถ้ารูของ loop ดูเหมือนตัวอย่างเหล่านี้ loop ของคุณยังไม่เจ๋งเท่าภาพด้านบน:
