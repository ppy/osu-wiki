# NAT 会议

**NAT 会议**是由[审核评估团队](/wiki/People/Nomination_Assessment_Team) (*NAT*) 在 [osu! Discord 服务器](/wiki/Community/Discord_servers#official)的 `#modding` 频道中主持的专题讨论。会议主题涉及[作图](/wiki/Beatmapping)和[摸图](/wiki/Modding)生态，包括存在的问题、潜在解决方案、新想法以及进度报告。尽管讨论的主要参与者多为 NAT 成员或[谱面审核成员 (BN)](/wiki/People/Beatmap_Nominators)，但事实上任何人都可以参与其中。

会议通常提前一周以上在 `#modding` 频道发布预告。

### 计划中的议题

- 编写 BN 手册/指南（应优先完成 NAT 手册）
- BN 活跃度/重置过审次数的量化考核（待提案审议）
- 重置过审的严重程度是否/如何/何时适用于谱面整体质量

### 已搁置的议题

这些议题因各种原因难以召开会议讨论，但未来可能成为可行议题：

| 议题 | 搁置原因 |
| :-- | :-- |
| BN/NAT 的工作需要减少机械感 | 需被分解为具体问题和建议 |
| 为 BN/NAT 工作提供有意义的奖励 | 不在 NAT 可控范围内 |
| 让 BN 参与对申请的评估和/或在职 BN 评估 | 范围过大，应首先提出书面提案 |
| NAT 的领导/交接/降职机制 | 范围过大，应首先提出书面提案 |
| 为作图/摸图 wiki 的清理/重写打下基础 | 愿意协助的人太少 |
