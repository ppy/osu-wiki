# 元数据

## 常规

### 规定

- **歌曲元数据（歌曲信息）必须参考[主要元数据来源](/wiki/Beatmap/Primary_metadata_source)。** 除非谱面上架标准允许或要求，否则不得修改主要来源的元数据。若无主要来源可用，则采用最广为人知的版本。
- **谱面中所有难度的 `Title`、`Artist`、`Tag`、`Source`、`BeatmapSetID` 字段必须相同。**
- **`Artist` 或 `Title` 字段超过了 81 个字符时必须缩短。**
  - 首先移除附加的标记。
    - 当艺术家字段很长、使用 `CV` 格式[^character-voice-actor]时，应移除角色名，仅列出 `声优` 的姓名。
  - 如果略去标记还是过长，则截断字段，并在合理位置添加 `...` 以示省略。
  - 被略去的信息必须添加到标签字段 (`Tags`) 内。
- **`Tags` 字段超过 1000 个字符时必须缩短.**
  - 首先移除非必要标签。
  - 尽可能保留玩家名等必要标签。
  - 其余标签优先保留与搜索最相关的部分。

### 准则

- **不应将商标作为标题或艺术家大小写的参考。** 因为商标通常是艺术化处理过的。除非有其他文本元数据佐证，否则应使用常规的标题大小写[^title-case]。
- **当翻唱或重混版的标题相较于原曲存在疑似拼写错误，如果并非刻意的艺术化风格，则应使用原曲拼写。** 若无法判断，应发起[讨论](/wiki/Beatmap_discussion#discussions)以达成共识。以下以原标题为 `triangles` 的重混音作品为例：
  - `triangls` 应纠正为 `triangles`
  - `triANGELS (angelic remix)` 应保持原文
  - `3angle5` 应保持原文
  - `Triangles` 可保留 `Triangles` 或改为 `triangles`
- **如果多份元数据均可用：**
  - 尽量符合已上架的谱面元数据。优先采用最新、最常见的用法，同时核实其正确性，并视需修正。
    - 如果艺术家并未故意使用别名，则也应尽量保持一致。
    - 若为适配标题与艺术家的风格化书写，可自由调整标记的大小写。
  - 罗马化字段优先采用官方罗马化方案或官方翻译，这样就能让它们容易被搜索和辨识。
  - 若选项存在冲突，应讨论并选取最佳方案。

### 准许

- **对于重混音、翻唱、演出作品：**
  - **艺术家字段可使用原曲艺术家，但标题字段必须注明此曲并非原版。** 该标记应置于括号内，并包含重混/翻唱艺术家或演奏者及版本描述。比如，由 `cYsmix` 作曲，由 `mocha4life` 翻唱的歌曲 `triangles` 可以格式化为 `cYsmix - triangles (mocha4life Cover)`。
  - **如果谱面创建者 (host) 就是歌曲的艺术家，则标题可以任意修改。**

## 符号

### 规定

- **如果标题或艺术家中含有出现两次、用于将部分字段分成一个组的符号，则符号前后与其他字段相邻的位置必须添加空格。** 比如，`歌曲★标题★`应修改为`歌曲 ★标题★`。

- **如果以下 Unicode 符号子集可以罗马化时，应该在前后添加空格：**
  - [补充箭头-A](https://zh.wikipedia.org/zh-cn/%E8%A1%A5%E5%85%85%E7%AE%AD%E5%A4%B4-A)、[箭头补充-B (Supplemental Arrows-B)](https://zh.wikipedia.org/zh-cn/%E8%A1%A5%E5%85%85%E7%AE%AD%E5%A4%B4-B)、[杂项符号和箭头](https://zh.wikipedia.org/zh-cn/%E6%9D%82%E9%A1%B9%E7%AC%A6%E5%8F%B7%E5%92%8C%E7%AE%AD%E5%A4%B4)
  - [什锦符号](https://zh.wikipedia.org/zh-cn/%E4%BB%80%E9%94%A6%E7%AC%A6%E5%8F%B7%20(Unicode%E5%8C%BA%E6%AE%B5))
  - [杂项符号](https://zh.wikipedia.org/zh-cn/%E6%9D%82%E9%A1%B9%E7%AC%A6%E5%8F%B7%20(Unicode%E5%8C%BA%E6%AE%B5))

  若艺术家有意使用符号且不暗示空格，则不适用此规则。比如，`。✰302？ionwan2go✰。`应该罗马化成`.*302?ionwan2go*.`，而不是`. * 302 ? ionwan2go * .`。其他字符集则应根据具体情况处理。

- **在罗马化字段，特殊符号应被转换为最接近的 [ASCII](https://en.wikipedia.org/wiki/ASCII) 可打印等效字符，或直接移除。**  下表列出了遇到这些符号时建议采用的罗马化方式。任何其他符号需根据具体情况讨论处理。

  | 符号 | 推荐的罗马化方案 |
  | :-- | :-- |
  | `★ ☆ ⚝ ✪ ✻` 或类似的星号 | `*` |
  | `♥ ♡` 或类似的心号 | `<3` |
  | `「 」『 』` | `""` |
  | `…` | `...` |
  | `。` | `.` |
  | `→` 或类似的右箭头 | `->` 或者 `-->` |
  | `←` 或类似的左箭头 | `<-` 或者 `<--` |
  | `《》` | `< >`、`<< >>`、或者 `""` |
  | `【】` | `""`、`()` 或者  `[]` |
  | `≠` | `=/=` 或者 `!=` |
  | `・` | `.`、`,` 或者 ` ` |
  | `×` | `x` |

## 艺术家 (Artist)

以下要点也适用于标题字段中出现的任何艺术家署名。

### 规定

- **不要将虚拟角色姓名或者项目名当做歌曲的唯一艺术家**，除非这个名字被艺术家用于别名。如果并不能追溯到真实存在的人，使用 `Unknown Artist`。
- **使用 `vs.`、`feat.`、`CV:` 等标记署名时，标记后必须跟一个空格。** 如果这些标记前还有其他文字，也需要在之前添加空格。
- **逗号之后必须添加空格。** 除非艺术家故意设计不留空格。

#### 规定：标记

当整个字段均为大写或小写时，标记可使用替代大小写以匹配字段其余部分。

- `vs.`
  - 任何形式的 `vs`、`versus`、`Vs` 等用于表示多位艺术家合作的标记必须重写为 `vs.`。
- `feat.`
  - 任何形式的 `feat`、`ft.`、`featuring`、`Feat.` 等用于表示特邀或客串艺术家的标记必须重写为 `feat.`。
- `角色名 (CV: 声优名)` 和 `角色名 (VO: 配音员名)`[^character-voice-actor]
  - 当动漫角色被标为歌曲的歌手时，使用这种格式。
  - 诸如 `c.v.`、`CV.`、`~cv~` 等类似的标记，必须使用这种格式替代。
  - 对于现场表演，仅署配音员姓名即可。

### 准则

- **对于各种同人圈子，你应该使用以下任一选项作为艺术家：**
  - `圈子名称`
  - `参与歌曲制作的知名成员`
  - `圈子名称 feat. 同人圈成员/外部贡献者/歌手`

  歌曲的外部贡献者通常列在致谢名单中，专门为这首歌列出的其他同人圈成员也应包括在内。至于参与歌曲制作的成员是否足够知名，可以成为唯一的艺术家，应该根据具体情况处理。

### 准许

- **如果参与制作歌曲的多位艺术家并未按照某种清晰的格式来展示，你以下允许项之一将他们组合在同一个艺术家字段中：**
  - `作曲人 feat. 歌手`
  - **艺术家姓名可直接使用 `,`、`&`、`x`、`/` 等符号连接。**
  - **若合并后的艺术家字段过长且无官方组合名称，可在艺术家字段中使用描述性标签代替。** 比如，`pippi, Mocha, Yuzu, Mani & Mari, Aiko, Alisa, Chirou, Taikonator, HitCircle, Fruit, Tama` 可以简化为 `osu! cast`。若无合适描述，则使用 `Various Artists`。
- **当角色声优以角色身份演唱歌曲时，可以使用 `角色名 (CV: 声优名)` 的格式。**

## 标题 (Title)

### 规定

- **当歌曲由两首或更多歌曲制作而成，必须执行以下任一操作：**
  - 将所有原曲标题列出，并使用 `,`、`&`、`x`、`/` 等作为分隔符。
    - 歌曲混搭也可以使用 `vs.` 来分隔不同的原曲歌标题。
  - 创建能描述曲目内容的自定义标题。例如将 `Pippi` 的多首原创曲目组合为一个[歌曲混编](/wiki/Beatmap/Song_compilation)时，使用 `Pippi's Original Songs Compilation` 作为标题。

### 标记

#### 规定

- **一首歌曲的非官方版本如果在内容、结构顺序和长度上都与官方版本相契合，则视为官方版本，并须添加合适的标记。这只适用于歌曲音频几乎无法与官方版本的音频分辨开的情况。**

##### 规定：在符合条件时，必须添加的标记

- **无版本标记但符合以下任一标记类别的歌曲，必须在标题末尾添加对应标记。**
- **已有版本标记的歌曲，则必须完全替换为以下列出的标准标记。**
- `(TV Size)`
  - 当歌曲特定版本用于 TV 节目（除了演唱会）、网络剧或非电影院首映的影视作品系列时，在标题尾部添加这个标签。
- `(Cut Ver.)`
  - 当歌曲为未经官方认可的剪辑版时使用。
  - 若剪辑版本为循环曲目的一个完整循环（如电子游戏或电影背景音乐），则可以不添加这个标记。
- `(Extended Edit)`
  - 当歌曲是非官方的延长版[^audio-rc-note]时使用。
- `(Sped Up Ver.)`、`(Nightcore Mix)`
  - 当歌曲的节奏被编辑加快时使用。
  - 只有歌曲的音调也被编辑加高时，才可以使用 `(Nightcore Mix)`，否则使用 `(Sped Up Ver.)`。
- **组合标记**
  - 如果编辑包含了加速*和*裁切，使用 `(Sped Up & Cut Ver.)` 或者 `(Nightcore & Cut Ver.)`。
  - 经过合适的可行性讨论之后，也可以使用其他组合标记。

##### 规定：必须标准化，但不一定要添加的标记

- **如果标题包含以下类似的标记，则必须完全替换为以下列出的对应标记。**
- **如果标题并未包含以下的标记，并且这首歌曲有多个具有相同原始元数据的版本时，在较短版本中添加一符合以下条件的标记。** 这有助于指示正确的版本。
- **若曲目是某首歌的唯一已知版本，则不得添加此类标记。**
- `(Short Ver.)`
  - 当歌曲同时存在长版和短版时，用于标记较短版本。诸如 `-Short Ver-`、`Short`、`~Short Version~` 的标记需要被替换掉。
- `(Game Ver.)`
  - 用于标记歌曲的游戏版本。诸如 `~Game Size~`、`(Game Size)`、`game OP edit`、`OP Version` 的标记需要被替换掉。
- `(Movie Ver.)`
  - 用于标记歌曲的电影版本。诸如 `Movie EDIT`、`~movie size~`、`Movie Cut`、`(Movie Version)` 的标记需要被替换掉。

#### 准则

- `(#### Ver.)`
  - 如果标题已经含有以上未列出的长度或版本标记，应将其改为描述性的 `(#### Ver.)` 标记，并使用标题大小写[^title-case]。比如：
    - `(Extended Version)` -> `(Extended Ver.)`
    - `(Long)` -> `(Long Ver.)`
  - 如果长度或版本标记已经被高度艺术化，甚至与标题合为一体时，可以忽略以上准则。比如 `Pippiquest (Pippi x Mocha Romantic Movie Remix Edition)`。

#### 准许

- **若歌曲标题其余部分具有统一的风格化大小写，标记可采用与之匹配的替代大小写。**
- **现场表演版本可以添加特殊的标记，比如 `(Live Ver.)`。** 也可以使用含有描述的标记，比如 `(2020 Tour Live Ver.)`。
- **如果以上列出的格式化标记方案可能含有歧义，也可以不添加标记，或根据具体情况使用自定义标记。** 此情况下需发起讨论以确定标记，并公开讨论结果。

## 来源 (Source)

### 规定

- **如果一首歌曲...则必须填写`来源`字段：**
  - **直接来自或与媒体（电子游戏、电视剧等）相关。专辑和托管网站除外。**
  - **是对源自或与某媒体作品相关联的歌曲进行的重混、编曲或翻唱，专辑和托管网站除外。**
  - **是特别为 osu! 制作的，比如 [osu! 原创曲](/wiki/osu!_originals)。** 除此之外，任何歌曲均*不允许*使用 `osu!` 作为来源字段。
  - **是特别为某一活动制作的，比如 `osu! World Cup` 等锦标赛或音乐会。**
- **如果一首歌曲包含或重混了多首歌曲，而这些歌曲并不能被囊括进同一个来源时，来源字段必须留空，并将这些来源添加至标签中。
- **来源字段必须足够精确。** 使用最细化的来源，而不是常规系列或项目名称。除非同一系列内的多个来源均适用。

### 准则

- **如果一首歌曲...**
  - **先被发行，之后才被某媒体作品采用或与之关联，则来源字段为可选。**
  - **出现在多个媒体内，此时任何符合条件的媒体都可作为来源字段。**
- **网站名称仅当满足以下条件时方可作为有效来源：**
  - **歌曲与网站关联于特定的文化现象，如 `Newgrounds` 等。**
  - **歌曲是作为网站主题曲或背景音乐而创作的。**

## 标签 (Tags)

### 规定

- **标签必须与谱面相关。** 比如描述谱面风格、歌曲、故事板、视频或背景内容。必须避免使用误导性的标签。
- **如果填入标签的姓名的字母之间含有空格，必须添加下划线。比如 `-[M o c h a]-` 应该改为 `-[M_o_c_h_a]-`。**
- **当符合条件时，标签必须包含以下内容：**
  - **[客串难度 (GD)](/wiki/Beatmap/Guest_difficulty) 谱师、故事板作者、皮肤作者、音效作者的玩家名。**
  - **如果歌曲在[精选艺术家目录](https://osu.ppy.sh/beatmaps/artists)中列出，则必须在标签中添加 `Featured Artist`。** 如果歌曲并未在此目录中列出，不得使用此标签。
  - **至少一个曲风标签和一个语言标签。**
    - 对于纯器乐歌曲，应使用 `instrumental` 作为语言标签。
    - 如果歌曲含有人造语言，需要在标签中添加 `conlang`，并将这种人造语言的名称作为语言标签。
    - 如果歌曲中的歌词没有实际意义，则不强制要求语言标签。
    - 若曲风和语言不明确，可添加尽可能多的适用标签。例如，使用多种语言演唱、横跨多种风格的歌曲。
  - **若歌曲为重混、编辑版、翻唱等，且原始标题和艺术家名未出现在其他字段中，须添加至此。**

### 准则

- **重混音、重编曲或歌曲混搭，应该在标签中同时列出其特定风格及原曲风格。** 如果一首 `Anime` 歌曲被重混音成了 `Electronic` 风格，则两个风格均应填入标签中。
- **当符合条件时，标签应该包含以下内容：**
  - **当存在多个元数据选项时，被略去而未在主要的字段中使用的艺术家、标题、来源信息。**
  - **其他与歌曲有关，但未列在艺术家字段中的艺术家，比如作曲、作词、吉他手等。**
  - **专辑、迷你专辑 (EP)、单曲专辑的名称。** 如果歌曲被多张专辑收录，只需列出一张。
  - **元数据中难以书写的部分的易搜索版本。**
    - 诸如 `don't`、`you're`，应该在标签中添加 `dont`、`youre`。
    - 对于 `cYsm1X` 创作的 `3angle5`，应该在标签中添加 `triangles` 和 `cYsmix`。
    - 美式英语和英式英语的不同，例如 `color` 和 `colour`。

### 准许

- **若其他元数据字段中已存在的词语是更长术语或短语的一部分，可在标签中重复出现。**

## 罗马化

### 规定

- **任何罗马化规定、准则和准许仅限于你自己罗马化时的情况。如果你使用了官方罗马化方案或官方翻译，则必须原封不动地使用，并填入罗马化字段。**
- **艺术家姓名必须依照其在 Unicode 字段中的排列顺序进行罗马化。**
- **若要罗马化外来词，则应使用其原词的拼写。**
- **当歌曲的标题或艺术家名使用了语义重复的单词，并且一部分是 Unicode 字符，另一部分是罗马化字符时，则罗马音字段必须仅使用罗马化版本，并移除重复的单词。** 比如，`ソレイユ -Soleil-` 的罗马化应该是 `Soleil`，`ピポピポ -People People-` 的罗马化应该是 `People People`。

### 规定：语言和书写系统罗马化

- **日语**
  - 使用[修正平文式罗马字方案](https://zh.wikipedia.org/wiki/%E5%B9%B3%E6%96%87%E5%BC%8F%E7%BD%97%E9%A9%AC%E5%AD%97)。
  - 除非歌曲或艺术家为了艺术化，混用了大小写（例如全大写或全小写的英文单词）之外，遵守标题大小写[^title-case]规定。
  - 罗马化 `ā` 为 `aa`、`ū` 为 `uu`、`ē` 为 `ee`、`ī` 为 `ii`
  - 根据日语原读法为 `おお` 或 `おう`，`ō` 分别罗马化为 `oo` 或者 `ou`。
  - 更多关于平文式罗马字的参考，请参阅[日文罗马字表](https://www.loc.gov/catdir/cpso/romanization/japanese.pdf)。
- **汉语**
  - 除专有名词外，必须将每个字逐个罗马化、首字母大写、并且在字与字之间添加空格。
  - 忽略拼音声调标识
  - 普通话：使用[汉语拼音方案](https://zh.wikipedia.org/zh-cn/%E6%B1%89%E8%AF%AD%E6%8B%BC%E9%9F%B3)
  - 广东话：使用[粤语拼音方案](https://baike.baidu.com/item/%E7%B2%A4%E8%AF%AD%E6%8B%BC%E9%9F%B3)
  - 其他方言：如何处理由谱师自行决定，建议咨询母语者。
- **西里尔字母**
  - 使用 [BGN/PCGN 系统](https://zh.wikipedia.org/zh-cn/BGN/PCGN%E7%BD%97%E9%A9%AC%E5%8C%96%E7%B3%BB%E7%BB%9F)。
  - `Е` 和 `е`：若单独出现，或位于 `a`、`e`、`ё`、`и`、`о`、`у`、`ы`、`э`、`ю`、`я`、`й`、`ъ`、`ь` 之后，罗马化为 `ye`。其他情况下罗马化为 `e`。
  - `ё`：如果出现在 `ж`、`ч`、`ш`、 `щ` 之后，罗马化为 `o`。其他情况罗马化为 `yo`。
  - 如果 `е` 作为 `ё` 的艺术化替代使用，此时也可以罗马化为 `yo` 。只要任一元数据字段中存在 `ё`，其替代拼写均应添加至标签 (`Tags`)。
  - 其他字符的罗马化，请参考[此文档第一页](https://assets.publishing.service.gov.uk/government/uploads/system/uploads/attachment_data/file/1116602/ROMANIZATION_OF_RUSSIAN_2022_final.pdf)。
  - 忽略该文档中的任何其他规定，因为这些规定要么无关紧要，要么与游戏无关。
- **北欧诸语**
  - 罗马化 `æ` 为 `ae`、`ø` 为 `oe`、`å` 为 `aa`.
- **瑞典语**
  - 罗马化 `ö` 为 `o`、`ä` 为 `a`、`å` 为 `a`
- **芬兰语**
  - 罗马化 `ö` 为 `o`、`ä` 为 `a`
- **德语**
  - 罗马化 `ü` 为 `ue`、`ö` 为 `oe`、`ä` 为 `ae`、`ß` 为 `ss`
- **艺术化字符**
  - 作为罗马字符艺术化替代的特殊字符，应根据上下文进行转换，例如其意图表达的单词拼写。比如，希腊字母 `β` 读作 `v`，但在歌曲 `βiοs` 中，应罗马化为 `Bios`。
- **其他未提及的语言或拼写系统**
  - 使用常见、容易辨识的罗马化系统。建议咨询母语者。

### 准许

- **如果被罗马化的语言本身不含空格，若空格用于清晰分隔项目，可将其罗马化为逗号。**

## 备注

[^title-case]: 所有主要单词的首字母需要大写，诸如 `and`、`to`、`or` 这样的连词和诸如 `the`、`a`、`an` 这样的冠词保持小写。
[^character-voice-actor]: `CV` (*character voice*) 用于表示歌曲由角色声优以角色身份演唱。`VO` (*voice over*) 用于表示歌曲由非角色声优的其他配音员以角色身份演唱。
[^audio-rc-note]: 对于玩家自制的延长版和歌曲混编，[通用音频标准](/wiki/Ranking_criteria#准则.2)也适用。
